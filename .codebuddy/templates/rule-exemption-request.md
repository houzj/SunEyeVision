## 审批流程

### 1. 申请人填写表格
- [ ] 完整填写豁免申请表
- [ ] 提供详细的原因和影响评估
- [ ] 说明替代方案和缓解措施

### 2. 提交技术负责人审批
- [ ] 通过邮件或代码审查提交申请
- [ ] 提供相关代码和文档链接
- [ ] 等待审批结果

### 3. 审批决策
- [ ] **批准**：可以豁免规则，遵守豁免期限
- [ ] **有条件批准**：需要调整后才能豁免
- [ ] **拒绝**：不能豁免，需要遵守规则或修改方案

### 4. 定期评估
- [ ] 在豁免期限内定期评估豁免必要性
- [ ] 如无需豁免，提前结束豁免
- [ ] 如需延长，重新提交申请

## 常见问题

### Q1: 什么情况下可以申请豁免？
**A**: 只有在以下情况下可以申请豁免：
1. 技术上无法遵守规则
2. 遵守规则会导致严重问题
3. 有充分的替代方案和风险控制措施

### Q2: Critical 和 High 规则可以豁免吗？
**A**: 可以，但需要更严格的审批和更充分的理由。

### Q3: 豁免期限可以延长吗？
**A**: 可以，但需要重新提交申请，说明延长的原因。

### Q4: 豁免结束后如何处理？
**A**: 豁免结束后，代码需要恢复正常，遵守规则要求。

### Q5: 多个规则需要豁免怎么办？
**A**: 为每个规则分别提交豁免申请，或在同一申请中列出多个规则。
