# JSON 序列化配置不一致问题修复报告

## 📋 修复概述

**修复日期**: 2026-03-15

**问题 ID**: JSON-001  
**严重级别**: 🔴 Critical

## 🎯 修复内容

### 修复的文件

#### 1. SolutionManager.cs

**文件路径**: `src/Workflow/SolutionManager.cs`

**修复位置**: LoadConfig 方法（第 667 行）

**问题**: 反序列化时没有配置 `PropertyNamingPolicy`，导致 JSON 文件中的 `camelCase` 属性名无法映射到 C# 的 `PascalCase` 属性名。

**修复代码**:
```csharp
// 修复前
var config = System.Text.Json.JsonSerializer.Deserialize<SolutionManagerConfig>(json);

// 修复后
var options = new System.Text.Json.JsonSerializerOptions
{
    PropertyNamingPolicy = System.Text.Json.JsonNamingPolicy.CamelCase
};
var config = System.Text.Json.JsonSerializer.Deserialize<SolutionManagerConfig>(json, options);
```

**影响**: 解决了配置文件无法加载的问题，确保 `KnownSolutions`、`CurrentSolution` 等配置能够正确恢复。

#### 2. JsonConfigManager.cs

**文件路径**: `src/Core/Services/JsonConfigManager.cs`

**修复位置**: LoadFromFileAsync 方法（第 105 行）

**问题**: 反序列化时没有配置 `PropertyNamingPolicy`，与 `SaveToFileAsync` 中的配置不一致。

**修复代码**:
```csharp
// 修复前
var loaded = JsonSerializer.Deserialize<Dictionary<string, string>>(json);

// 修复后
var options = new JsonSerializerOptions
{
    PropertyNamingPolicy = JsonNamingPolicy.CamelCase
};
var loaded = JsonSerializer.Deserialize<Dictionary<string, string>>(json, options);
```

**影响**: 确保通用配置管理器能够正确加载配置文件，避免数据丢失。

## 🔧 根本原因分析

### 技术原因

**System.Text.Json 序列化配置不一致**：

| 操作 | Serialize | Deserialize |
|------|-----------|-------------|
| **配置** | `PropertyNamingPolicy = JsonNamingPolicy.CamelCase` | 无配置 |
| **JSON 格式** | `knownSolutions` (camelCase) | 期望 `KnownSolutions` (PascalCase) |
| **结果** | 文件正确保存 | 无法正确加载 |

### 根本原因

1. **命名策略不匹配**
   - 序列化使用 `camelCase` 命名
   - 反序列化期望 `PascalCase` 命名
   - 属性名无法正确映射

2. **配置不一致**
   - `SaveConfig` 和 `LoadConfig` 使用了不同的 `JsonSerializerOptions`
   - `SaveToFileAsync` 和 `LoadFromFileAsync` 使用了不同的 `JsonSerializerOptions`

3. **静态检查缺失**
   - 没有单元测试验证序列化/反序列化一致性
   - 没有静态分析工具检测此类问题
