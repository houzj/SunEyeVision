## ✅ 阶段二修复工作

### 修复1: DataMigrationService.cs

**文件**: `src/Workflow/DataMigrationService.cs`

**问题**:
- LoadOldSolutionFile方法使用了不完整的JsonSerializerOptions
- 缺少WriteIndented、DefaultIgnoreCondition、Encoder配置

**修复前**:
```csharp
var options = new JsonSerializerOptions
{
    PropertyNameCaseInsensitive = true,
    PropertyNamingPolicy = JsonNamingPolicy.CamelCase
};
```

**修复后**:
```csharp
return JsonSerializer.Deserialize<OldSolutionFile>(json, JsonSerializationOptions.Compatibility);
```

**理由**: 数据迁移需要兼容性配置（PropertyNameCaseInsensitive = true）

### 修复2: LayoutConfig.cs

**文件**: `src/UI/Models/LayoutConfig.cs`

**问题**:
- Save方法使用了简化的JsonSerializerOptions（只有WriteIndented）
- Load方法没有指定options（使用默认配置）
- 序列化和反序列化配置不一致

**修复前**:
```csharp
// Save方法
var json = JsonSerializer.Serialize(this, new JsonSerializerOptions
{
    WriteIndented = true
});

// Load方法
var config = JsonSerializer.Deserialize<LayoutConfig>(json);
```

**修复后**:
```csharp
// Save方法
var json = JsonSerializer.Serialize(this, JsonSerializationOptions.Default);

// Load方法
var config = JsonSerializer.Deserialize<LayoutConfig>(json, JsonSerializationOptions.Default);
```

**理由**: UI配置文件需要格式化输出，使用Default配置

### 修复3: PluginManager.cs

**文件**: `src/Core/Services/PluginManager.cs`

**问题**:
- LoadPluginMetadata方法没有指定JsonSerializerOptions

**修复前**:
```csharp
return JsonSerializer.Deserialize<PluginMetadata>(json) ?? new PluginMetadata();
```

**修复后**:
```csharp
return JsonSerializer.Deserialize<PluginMetadata>(json, JsonSerializationOptions.Default) ?? new PluginMetadata();
```

**理由**: 插件元数据需要标准配置
