# 区域序列化问题诊断日志追踪指南

## 问题描述
用户在区域编辑器中绘制了检测区域，但保存解决方案后，JSON 文件中 `InspectionRegions.Regions` 为空数组 `[]`。

## 诊断策略
通过在关键位置添加日志，追踪数据从绘制到序列化的完整链路。

## 日志链路追踪

**预期**：当打开调试窗口时，记录参数注入时的状态。

## 常见问题诊断

### 问题 1：共享引用未建立
**症状**：`SetRegionsSource` 日志显示 `引用相同=False`

**可能原因**：
1. `Parameters` 依赖属性变化时，`OnRegionsChanged` 没有被触发
2. XAML 绑定的 DataContext 没有正确更新

**检查**：
- 查看 `[OnParametersChanged]` 日志是否记录了参数变化
- 查看 `[Regions.Setter]` 和 `[OnRegionsChanged]` 日志是否按顺序出现

### 问题 2：区域添加到了错误的集合
**症状**：绘制后 `[FinishInteraction]` 日志显示 `TotalRegions=1`，但保存时 `Regions.Count=0`

**可能原因**：
1. 共享引用建立后，集合被替换
2. 绘制时使用的集合和最终序列化的集合不是同一个

**检查**：
- 对比 `SetRegionsSource` 中的 `Regions.HashCode` 和保存时的 `Regions.HashCode`
- 如果不同，说明集合被替换过

### 问题 3：反序列化覆盖了数据
**症状**：第一次保存正常，关闭后重新打开，区域丢失

**可能原因**：
1. `RegionCollectionParameter.InspectionRegions` 有 setter
2. JSON 反序列化时创建了新的空集合

**检查**：
- 查看 `[RegionCollectionParameter.Regions.Getter]` 日志中的 HashCode
- 如果每次保存时 HashCode 都不同，说明每次都创建了新的集合对象

## 测试步骤

1. **编译项目**：确保所有日志代码已编译

2. **打开调试窗口**：
   - 添加阈值化工具
   - 打开调试窗口
   - 检查日志中 `[OnParametersChanged]` 和 `[OnRegionsChanged]` 是否按顺序出现

3. **绘制区域**：
   - 使用矩形/圆形工具绘制一个检测区域
   - 检查日志中 `[FinishInteraction]` 是否显示区域已添加

4. **保存解决方案**：
   - 保存解决方案
   - 检查日志中 `[SaveSolutionData]` 是否显示 `Regions.Count>=1`
   - 检查日志中 `InspectionRegions JSON` 是否包含区域数据

5. **重新加载并验证**：
   - 关闭软件
   - 重新打开
   - 加载保存的解决方案
   - 打开调试窗口
   - 检查区域是否正确恢复

## 日志分析方法

### 使用 Filter 过滤日志
在日志显示器中，使用以下关键词过滤：
- `ThresholdTool` - 查看阈值工具相关日志
- `RegionEditorControl` - 查看区域编辑器控制日志
- `RegionEditor` - 查看区域编辑器视图模型日志
- `MainWindowViewModel` - 查看保存相关日志
- `SolutionRepository` - 查看序列化相关日志
- `RegionCollectionParameter` - 查看集合参数相关日志

### 对比 HashCode
关键是要对比不同阶段的 HashCode：
- `Parameters.HashCode` - 参数对象的标识
- `InspectionRegions.HashCode` - 检测区域集合容器的标识
- `Regions.HashCode` - 实际区域列表的标识

如果 HashCode 不一致，说明对象引用不同，数据没有同步。
