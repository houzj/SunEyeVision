# JSON 序列化配置不一致问题分析与优化方案

## 📋 问题概述

**问题 ID**: JSON-001  
**发现日期**: 2026-03-15  
**严重级别**: 🔴 Critical  
**影响范围**: 配置持久化功能  

### 问题描述

在解决方案管理系统的配置持久化中，出现序列化和反序列化配置不一致的问题，导致配置文件无法正确加载，表现为：
- 配置文件能够正常保存
- 重启程序后配置数据丢失
- 日志显示配置文件被读取，但加载的数据为空

## 🔍 根本原因分析

### 1. 核心问题

**System.Text.Json 序列化/反序列化选项不一致**

在多个类中，`Serialize` 和 `Deserialize` 使用了不同的 `JsonSerializerOptions` 配置：

#### 案例 1: SolutionManager.cs (已修复)

**Serialize 方法 (SaveConfig - 第 719 行)**:
```csharp
var options = new JsonSerializerOptions
{
    WriteIndented = true,
    PropertyNamingPolicy = JsonNamingPolicy.CamelCase  // ✅ 配置了小驼峰命名
};

var json = JsonSerializer.Serialize(config, options);
File.WriteAllText(_configFilePath, json);
```

**Deserialize 方法 (LoadConfig - 修复前第 667 行)**:
```csharp
var json = File.ReadAllText(_configFilePath);
var config = JsonSerializer.Deserialize<SolutionManagerConfig>(json);
// ❌ 没有配置 PropertyNamingPolicy，导致属性名无法映射
```

#### 案例 2: JsonConfigManager.cs (已修复)

**Serialize 方法 (SaveToFileAsync - 第 80 行)**:
```csharp
var options = new JsonSerializerOptions
{
    WriteIndented = true,
    Encoder = System.Text.Encodings.Web.JavaScriptEncoder.UnsafeRelaxedJsonEscaping
};

var json = JsonSerializer.Serialize(_configs, options);
// ❌ 缺少 PropertyNamingPolicy 配置
```

**Deserialize 方法 (LoadFromFileAsync - 修复前第 105 行)**:
```csharp
var loaded = JsonSerializer.Deserialize<Dictionary<string, string>>(json);
// ❌ 没有配置 PropertyNamingPolicy
```

### 2. 技术原理

#### System.Text.Json 默认行为

`System.Text.Json.JsonSerializer` 默认情况下：
- **序列化时**: 保留属性原本的大小写（PascalCase）
- **反序列化时**: 必须精确匹配属性名（区分大小写）

#### 命名策略的作用

```csharp
// 使用 PropertyNamingPolicy = JsonNamingPolicy.CamelCase

// C# 类定义 (PascalCase)
public class SolutionManagerConfig
{
    public List<SolutionInfo> KnownSolutions { get; set; }
    public string CurrentSolution { get; set; }
    public bool SkipStartupConfig { get; set; }
}

// 序列化后的 JSON (camelCase)
{
  "knownSolutions": [...],      // KnownSolutions -> knownSolutions
  "currentSolution": "",        // CurrentSolution -> currentSolution
  "skipStartupConfig": false    // SkipStartupConfig -> skipStartupConfig
}

// 反序列化时需要使用相同的命名策略，否则：
// - 有命名策略: "knownSolutions" -> KnownSolutions ✅
// - 无命名策略: "knownSolutions" -> 无法映射到 KnownSolutions ❌
```

### 4. 根本原因总结

| 维度 | Serialize | Deserialize | 结果 |
|------|-----------|-------------|------|
| **配置选项** | `PropertyNamingPolicy = JsonNamingPolicy.CamelCase` | 无配置 | ❌ 不一致 |
| **JSON 格式** | `knownSolutions` (camelCase) | 期望 `KnownSolutions` (PascalCase) | ❌ 无法映射 |
| **反序列化结果** | - | `config.KnownSolutions = null` | ❌ 数据丢失 |
| **日志表现** | 配置保存成功 | 配置加载完成（但数据为空） | ❌ 误导性日志 |

### 方案 1: 创建统一的序列化选项管理器

#### 1.1 设计目标

- **统一管理**: 全局唯一序列化选项配置
- **类型安全**: 提供强类型访问
- **性能优化**: 复用序列化选项实例
- **易于扩展**: 支持自定义配置

#### 1.2 实现方案

**创建 JsonSerializerOptionsProvider 类**:

```csharp
// src/Core/Services/Serialization/JsonSerializerOptionsProvider.cs
using System.Text.Json;
using System.Text.Json.Serialization;

namespace SunEyeVision.Core.Services.Serialization;

/// <summary>
/// JSON 序列化选项提供者
/// </summary>
/// <remarks>
/// 统一管理项目的 JSON 序列化选项，确保序列化和反序列化配置一致。
/// 
/// 使用示例:
/// var options = JsonSerializerOptionsProvider.Default;
/// var json = JsonSerializer.Serialize(obj, options);
/// var obj = JsonSerializer.Deserialize<T>(json, options);
/// </remarks>
public static class JsonSerializerOptionsProvider
{
    /// <summary>
    /// 默认序列化选项（标准配置）
    /// </summary>
    /// <remarks>
    /// 使用 camelCase 命名策略，格式化输出，忽略 null 值
    /// </remarks>
    public static readonly JsonSerializerOptions Default = new()
    {
        WriteIndented = true,
        PropertyNamingPolicy = JsonNamingPolicy.CamelCase,
        DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull
    };

    /// <summary>
    /// 紧凑序列化选项（无格式化）
    /// </summary>
    /// <remarks>
    /// 适用于需要最小化文件大小的场景，如日志、网络传输等
    /// </remarks>
    public static readonly JsonSerializerOptions Compact = new()
    {
        WriteIndented = false,
        PropertyNamingPolicy = JsonNamingPolicy.CamelCase,
        DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull
    };

    /// <summary>
    /// 调试序列化选项（包含类型信息）
    /// </summary>
    /// <remarks>
    /// 适用于开发调试，包含类型信息以便于调试
    /// </remarks>
    public static readonly JsonSerializerOptions Debug = new()
    {
        WriteIndented = true,
        PropertyNamingPolicy = JsonNamingPolicy.CamelCase,
        DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull,
        WriteIndented = true,
        IncludeFields = true
    };

    /// <summary>
    /// 自定义序列化选项
    /// </summary>
    /// <param name="writeIndented">是否格式化输出</param>
    /// <param name="propertyNamingPolicy">命名策略</param>
    /// <returns>自定义序列化选项</returns>
    public static JsonSerializerOptions Create(
        bool writeIndented = true,
        JsonNamingPolicy? propertyNamingPolicy = null)
    {
        return new JsonSerializerOptions
        {
            WriteIndented = writeIndented,
            PropertyNamingPolicy = propertyNamingPolicy ?? JsonNamingPolicy.CamelCase,
            DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull
        };
    }
}
```

#### 4.1 运行时验证

**在序列化/反序列化后添加验证逻辑**:

```csharp
public static class SerializationValidator
{
    /// <summary>
    /// 验证序列化结果是否包含预期的属性
    /// </summary>
    public static bool ValidateJsonContainsProperties<T>(T obj, string json)
    {
        var properties = typeof(T).GetProperties();
        var jsonLower = json.ToLower();
        
        foreach (var prop in properties)
        {
            // 将 PascalCase 转换为 camelCase
            var expectedName = char.ToLower(prop.Name[0]) + prop.Name.Substring(1);
            if (!jsonLower.Contains(expectedName.ToLower()))
            {
                return false;
            }
        }
        
        return true;
    }

    /// <summary>
    /// 验证反序列化结果是否丢失了数据
    /// </summary>
    public static bool ValidateDeserializedData<T>(T original, T? deserialized)
    {
        if (deserialized == null)
            return false;

        var properties = typeof(T).GetProperties();
        foreach (var prop in properties)
        {
            var originalValue = prop.GetValue(original);
            var deserializedValue = prop.GetValue(deserialized);
            
            if (!Equals(originalValue, deserializedValue))
            {
                return false;
            }
        }
        
        return true;
    }
}

# .github/workflows/serialization-tests.yml
name: Serialization Tests

on: [push, pull_request]

jobs:
  serialization-tests:
    runs-on: windows-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup .NET
      uses: actions/setup-dotnet@v3
      with:
        dotnet-version: '8.0.x'
    
    - name: Restore dependencies
      run: dotnet restore
    
    - name: Build
      run: dotnet build --configuration Release
    
    - name: Run serialization tests
      run: dotnet test --filter "FullyQualifiedName~Serialization"
    
    - name: Run round-trip tests
      run: dotnet test --filter "FullyQualifiedName~RoundTrip"
```
