# 日志系统竞态条件修复方案

## 问题描述

### 异常现象
在最近的工作流执行优化后，日志系统突然出现 `ArgumentOutOfRangeException` 异常：

```
System.ArgumentOutOfRangeException: 索引超出了数组界限。
   at System.Windows.Controls.ItemCollection.get_Item(Int32 index)
   at SunEyeVision.UI.Views.Controls.Panels.PropertyPanelControl.OnLogsUpdated(Object sender, EventArgs e) 行 417
```

### 问题触发条件
- 高频日志写入场景（每秒30-100+条日志）
- CircularBufferLogCollection达到容量上限（5000条）
- DataGrid执行重新绑定操作

## 根本原因分析

### 竞态条件链路

```
WorkflowExecutionEngine（高频日志）
    ↓
UILogWriter.AddToCollection
    ↓
CircularBufferLogCollection.AddRange（触发Reset事件）
    ↓
DataGrid重新绑定（Items被清空）
    ↓
LogsAdded事件同步触发
    ↓
PropertyPanelControl.OnLogsUpdated
    ↓
LogDataGrid.Items[Count-1] ← Count=0时抛出异常
```

### 设计缺陷

#### 1. 同步事件触发
UILogWriter在添加日志后立即同步触发事件：
```csharp
_logEntries.AddRange(entries);  // 触发CollectionChanged.Reset
LogsAdded?.Invoke(this, new LogBatchEventArgs(entries));  // 同步触发，不等待UI绑定完成
```

#### 2. 职责混淆
- CircularBufferLogCollection既是数据模型，又是事件源
- UILogWriter同时负责数据存储和事件分发
- 违反单一职责原则

#### 3. 长事件传播链
日志写入 → CollectionChanged → LogsAdded → LogDataScroll → UI更新
缺少中间缓冲和节流机制

#### 4. 紧耦合
PropertyPanelControl直接访问DataGrid.Items，缺乏UI抽象层

#### 5. 无节流机制
每条日志都触发完整的UI更新链路，高频场景下性能和稳定性问题突出

## 技术方案

### 整体架构

采用**生产者-消费者模式**重构日志系统，引入事件聚合层解耦日志生产和UI消费：

```
┌─────────────────┐
│ UILogWriter     │ 日志生产者
└────────┬────────┘
         │ 写入日志
         ↓
┌─────────────────┐
│ LogEventAggregator │ 事件聚合（节流、批处理）
└────────┬────────┘
         │ 批量更新（100ms间隔）
         ↓
┌─────────────────┐
│ CircularBuffer  │ 数据存储
│ LogCollection   │
└────────┬────────┘
         │ CollectionChanged
         ↓
┌─────────────────┐
│ LogPanelViewModel │ UI视图模型
└────────┬────────┘
         │ Binding
         ↓
┌─────────────────┐
│ DataGrid        │ UI展示
└─────────────────┘
```

**目标**：立即防止崩溃，为后续重构争取时间

**实施内容**：

1. **PropertyPanelControl.xaml.cs** 添加边界检查

```csharp
private void OnLogsUpdated(object? sender, EventArgs e)

    try
    {

            var lastItem = LogDataGrid.Items[^1];
            LogDataGrid.ScrollIntoView(lastItem);

    }
```

2. **CircularBufferLogCollection.cs** 延迟Reset触发
```csharp
private void RemoveOldItems(int count)
{
    _removedCount += count;
    // 延迟触发Reset，等待UI绑定完成
    _dispatcher.BeginInvoke(DispatcherPriority.ContextIdle, () =>
    {
        OnCollectionChanged(new NotifyCollectionChangedEventArgs(
            NotifyCollectionChangedAction.Reset));
        _removedCount = 0;
```

#### 阶段二：事件聚合器实现（2-3天）

**目标**：引入异步事件聚合机制，解决根本设计问题

**实施内容**：

1. **创建 LogEventAggregator.cs**

```csharp
using System;
using System.Collections.Concurrent;
using System.Collections.Generic;
using System.Threading;
using System.Windows.Threading;

namespace SunEyeVision.Core.Services.Logging
{
    /// <summary>
    /// 日志事件聚合器，用于节流和批处理高频日志事件
    /// </summary>
    public class LogEventAggregator
    {
        private readonly Dispatcher _dispatcher;
        private readonly ConcurrentQueue<LogEntry> _pendingLogs;
        private readonly object _lock = new();
        private bool _isSchedulingUpdate;
        private readonly TimeSpan _throttleInterval;
        private const int MaxBatchSize = 200;

        public event EventHandler<LogBatchEventArgs>? LogsAggregated;

        public LogEventAggregator(Dispatcher dispatcher, TimeSpan throttleInterval)
        {
            _dispatcher = dispatcher;
            _throttleInterval = throttleInterval;
            _pendingLogs = new ConcurrentQueue<LogEntry>();
        }

        /// <summary>
        /// 收集日志条目到待处理队列
        /// </summary>
        public void EnqueueLog(LogEntry entry)
        {
            _pendingLogs.Enqueue(entry);
            ScheduleUpdate();
        }

        /// <summary>
        /// 批量收集日志条目
        /// </summary>
        public void EnqueueLogs(IReadOnlyList<LogEntry> entries)
        {
            foreach (var entry in entries)
            {
                _pendingLogs.Enqueue(entry);
            }
            ScheduleUpdate();
        }

        /// <summary>
        /// 调度更新任务
        /// </summary>
        private void ScheduleUpdate()
        {
            lock (_lock)
            {
                if (_isSchedulingUpdate)
                    return;

                _isSchedulingUpdate = true;
                _dispatcher.BeginInvoke(DispatcherPriority.ContextIdle, () =>
                {
                    Thread.Sleep(_throttleInterval);
                    FlushLogs();
                });
            }
        }

        /// <summary>
        /// 批量刷新待处理的日志
        /// </summary>
        private void FlushLogs()
        {
            var batch = new List<LogEntry>(MaxBatchSize);

            lock (_lock)
            {
                // 从队列中提取日志
                while (_pendingLogs.TryDequeue(out var entry) && batch.Count < MaxBatchSize)
                {
                    batch.Add(entry);
                }

                _isSchedulingUpdate = false;

                // 如果还有日志，继续调度
                if (!_pendingLogs.IsEmpty)
                {
                    ScheduleUpdate();
                }
            }

            // 触发聚合事件
            if (batch.Count > 0)
            {
                LogsAggregated?.Invoke(this, new LogBatchEventArgs(batch));
            }
        }

        /// <summary>
        /// 清空待处理队列
        /// </summary>
        public void Clear()
        {
            while (_pendingLogs.TryDequeue(out _)) { }
        }
    }
}
```

3. **修改 UILogWriter.cs**

```csharp
public class UILogWriter : ILogWriter
{
    private readonly CircularBufferLogCollection _logEntries;
    private readonly Dispatcher _dispatcher;
    private readonly LogEventAggregator _eventAggregator;

    public event EventHandler<LogBatchEventArgs>? LogsAdded;

    public UILogWriter(Dispatcher dispatcher)
    {
        _dispatcher = dispatcher;
        _logEntries = new CircularBufferLogCollection(5000, dispatcher);
        _eventAggregator = new LogEventAggregator(dispatcher, TimeSpan.FromMilliseconds(100));

        // 订阅聚合器事件
        _eventAggregator.LogsAggregated += OnLogsAggregated;
    }

    public void Write(LogLevel level, string message, string source, Exception? exception = null)
    {
        var entry = new LogEntry(level, message, source, exception);
        AddToCollection(new[] { entry });
    }

    private void AddToCollection(IReadOnlyList<LogEntry> entries)
    {
        if (_dispatcher.CheckAccess())
        {
            // 使用聚合器代替直接添加和事件触发
            _eventAggregator.EnqueueLogs(entries);
        }
        else
        {
            _dispatcher.Invoke(() => AddToCollection(entries));
        }
    }

    private void OnLogsAggregated(object? sender, LogBatchEventArgs e)
    {
        // 批量添加到集合
        _logEntries.AddRange(e.Logs);

        // 触发外部订阅者
        LogsAdded?.Invoke(this, e);
    }

    public IReadOnlyList<LogEntry> GetLogs()
    {
        return _logEntries.ToList();
    }

    public void ClearLogs()
    {
        _logEntries.Clear();
        _eventAggregator.Clear();
    }
}
```

**预期效果**：
- 事件触发频率降低50-75%（从每条日志降低到每100ms批量）
- 消除同步事件导致的竞态条件
- UI更新更流畅，性能提升30-50%

#### 阶段三：架构优化（2-3天）

**目标**：优化整体架构，提高可维护性和扩展性

**实施内容**：

1. **优化 CircularBufferLogCollection.cs**

```csharp
public class CircularBufferLogCollection : IList<LogEntry>, INotifyCollectionChanged
{
    private readonly Dispatcher _dispatcher;
    private readonly int _capacity;
    private readonly Queue<LogEntry> _buffer;
    private readonly List<LogEntry> _view;
    private int _removedSinceLastReset;

    public event NotifyCollectionChangedEventHandler? CollectionChanged;

    public CircularBufferLogCollection(int capacity, Dispatcher dispatcher)
    {
        _capacity = capacity;
        _dispatcher = dispatcher;
        _buffer = new Queue<LogEntry>(capacity);
        _view = new List<LogEntry>(capacity);
    }

    public void AddRange(IReadOnlyList<LogEntry> entries)
    {
        var toAdd = new List<LogEntry>(entries.Count);

        foreach (var entry in entries)
        {
            if (_buffer.Count >= _capacity)
            {
                _buffer.Dequeue();
                _removedSinceLastReset++;
            }
            _buffer.Enqueue(entry);
            toAdd.Add(entry);
        }

        _view.AddRange(toAdd);

        // 批量触发Changed事件
        if (toAdd.Count > 0)
        {
            OnCollectionChanged(new NotifyCollectionChangedEventArgs(
                NotifyCollectionChangedAction.Add,
                toAdd,
                _view.Count - toAdd.Count));
        }

        // 延迟触发Reset
        if (_removedSinceLastReset > 0)
        {
            ScheduleReset();
        }
    }

    private void ScheduleReset()
    {
        var removedCount = _removedSinceLastReset;
        _removedSinceLastReset = 0;

        _dispatcher.BeginInvoke(DispatcherPriority.ContextIdle, () =>
        {
            OnCollectionChanged(new NotifyCollectionChangedEventArgs(
                NotifyCollectionChangedAction.Reset));
        });
    }

    protected virtual void OnCollectionChanged(NotifyCollectionChangedEventArgs e)
    {
        CollectionChanged?.Invoke(this, e);
    }
}
```

2. **优化 LogPanelViewModel.cs**

```csharp
public class LogPanelViewModel : ViewModelBase
{
    private readonly ILogWriter _logWriter;
    private readonly Dispatcher _dispatcher;
    private bool _isAutoScroll = true;
    private DateTime _lastScrollTime;

    public ObservableCollection<LogEntry> DisplayLogs { get; } = new();

    public LogPanelViewModel(ILogWriter logWriter)
    {
        _logWriter = logWriter;
        _dispatcher = Dispatcher.CurrentDispatcher;

        if (_logWriter is UILogWriter uiLogWriter)
        {
            uiLogWriter.LogsAdded += OnLogsAdded;
        }
    }

    private void OnLogsAdded(object? sender, LogBatchEventArgs e)
    {
        // 节流控制：每50ms最多更新一次UI
        var now = DateTime.UtcNow;
        if ((now - _lastScrollTime).TotalMilliseconds < 50)
        {
            return;
        }
        _lastScrollTime = now;

        _dispatcher.BeginInvoke(() =>
        {
            // 批量添加到显示集合
            foreach (var log in e.Logs)
            {
                DisplayLogs.Add(log);
            }

            // 保持显示数量在合理范围（最近1000条）
            while (DisplayLogs.Count > 1000)
            {
                DisplayLogs.RemoveAt(0);
            }

            // 自动滚动
            if (_isAutoScroll)
            {
                // 滚动逻辑由View层处理
            }
        });
    }

    public void ClearLogs()
    {
        DisplayLogs.Clear();
        _logWriter.ClearLogs();
    }
}
```

3. **优化 PropertyPanelControl.xaml.cs**

```csharp
public partial class PropertyPanelControl : UserControl
{
    private readonly DispatcherTimer _scrollTimer;

    public PropertyPanelControl()
    {
        InitializeComponent();
        _scrollTimer = new DispatcherTimer
        {
            Interval = TimeSpan.FromMilliseconds(200)
        };
        _scrollTimer.Tick += OnScrollTimerTick;
    }

    private void OnLogsUpdated(object? sender, EventArgs e)
    {
        // 使用定时器节流滚动操作
        if (!_scrollTimer.IsEnabled)
        {
            _scrollTimer.Start();
        }
    }

    private void OnScrollTimerTick(object? sender, EventArgs e)
    {
        _scrollTimer.Stop();

        try
        {
            if (LogDataGrid?.Items.Count > 0)
            {
                var lastItem = LogDataGrid.Items[^1];
                LogDataGrid.ScrollIntoView(lastItem);
            }
        }
        catch (ArgumentOutOfRangeException)
        {
            // 忽略临时同步问题
        }
    }
}
```

## 测试策略

### 单元测试

```csharp
[TestClass]
public class LogEventAggregatorTests
{
    [TestMethod]
    public void EnqueueLog_ShouldTriggerAggregatedEvent()
    {
        // Arrange
        var aggregator = new LogEventAggregator(Dispatcher.CurrentDispatcher, TimeSpan.FromMilliseconds(100));
        var triggered = false;

        aggregator.LogsAggregated += (s, e) =>
        {
            triggered = true;
            Assert.IsTrue(e.Logs.Count > 0);
        };

        // Act
        aggregator.EnqueueLog(new LogEntry(LogLevel.Info, "Test", "Source"));

        // Assert
        Thread.Sleep(200);
        Assert.IsTrue(triggered);
    }

    [TestMethod]
    public void EnqueueLogs_ShouldBatchEvents()
    {
        // Arrange
        var aggregator = new LogEventAggregator(Dispatcher.CurrentDispatcher, TimeSpan.FromMilliseconds(50));
        LogBatchEventArgs? capturedArgs = null;

        aggregator.LogsAggregated += (s, e) => capturedArgs = e;

        // Act
        var logs = Enumerable.Range(0, 300)
            .Select(i => new LogEntry(LogLevel.Info, $"Log {i}", "Source"))
            .ToList();
        aggregator.EnqueueLogs(logs);

        Thread.Sleep(200);

        // Assert
        Assert.IsNotNull(capturedArgs);
        Assert.IsTrue(capturedArgs.Logs.Count <= 200); // MaxBatchSize
    }
}
```

## 风险控制

### 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 异步逻辑复杂度 | 中 | 充分测试，保持简单的设计 |
| UI线程阻塞 | 高 | 使用ContextIdle优先级，批处理 |
| 内存泄漏 | 中 | 正确实现IDisposable，弱引用 |
| 向后兼容性 | 低 | 保持接口不变，内部重构 |

### 回退方案

如果新架构出现问题，可按以下顺序回退：

1. **阶段一回退**：仅保留PropertyPanelControl的边界检查
2. **阶段二回退**：禁用LogEventAggregator，恢复同步模式
3. **完整回退**：使用git恢复到重构前的稳定版本
