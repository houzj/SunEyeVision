# 图像切换优化集成说明

## 概述

已成功集成图像切换性能优化组件，目标是将图像切换延迟从50-150ms降低到<10ms（缓存命中时）。

## 已创建的核心组件

### 1. CachedExecutionResult.cs
**路径**: `src/UI/Services/Performance/CachedExecutionResult.cs`

**功能**: 缓存模型，存储执行结果
- 节点ID和图像源ID
- 处理后的图像（BitmapSource）
- 原始图像（可选）
- 执行时间统计
- LRU淘汰支持

**关键特性**:
- 自动计算缓存大小
- 支持过期检测（默认300秒）
- 提供访问时间更新

### 2. WorkflowEnginePool.cs
**路径**: `src/UI/Services/Performance/WorkflowEnginePool.cs`

**功能**: 工作流引擎对象池
- 避免频繁创建/销毁引擎实例
- 支持预热（Prewarm）
- Rent/Return模式
- 最大池大小限制（默认3个）

**使用示例**:
```csharp
var pool = new WorkflowEnginePool(maxPoolSize: 3);
var engine = pool.Rent();
try {
    // 使用引擎执行工作流
}
finally {
    pool.Return(engine);
}
```

### 3. PreExecutionCache.cs
**路径**: `src/UI/Services/Performance/PreExecutionCache.cs`

**功能**: 预执行缓存管理
- LRU淘汰策略
- 内存限制（默认100MB）
- 数量限制（默认50项）
- 自动过期清理（每60秒）

**关键特性**:
- 线程安全（ConcurrentDictionary）
- 自动计算缓存大小
- 支持内存和数量双重限制

### 4. ImageConverterPool.cs
**路径**: `src/UI/Services/Performance/ImageConverterPool.cs`

**功能**: 图像转换优化
- OpenCvSharp Mat → WPF BitmapSource
- WriteableBitmap直接内存写入
- 转换器对象缓存
- 支持灰度图、BGR、BGRA格式

**性能优化**:
- 避免中间格式转换
- 直接内存复制
- 复用WriteableBitmap对象

### 5. BatchUIUpdater.cs
**路径**: `src/UI/Services/Performance/BatchUIUpdater.cs`

**功能**: 批量UI更新
- 合并UI更新请求
- DispatcherTimer实现（默认16ms约60fps）
- 支持立即刷新（Flush）
- 单次最大处理数量限制（默认10项）

**使用示例**:
```csharp
var updater = new BatchUIUpdater(updateInterval: 16, maxBatchSize: 10);
updater.Enqueue(() => {
    // UI更新操作
});
updater.Flush(); // 立即执行所有待处理更新
```

### 6. WorkflowExecutionOrchestrator.cs
**路径**: `src/UI/Services/Performance/WorkflowExecutionOrchestrator.cs`

**功能**: 核心协调器，整合所有优化组件
- `ExecuteAndDisplayAsync()`: 带缓存优化的执行
- `PreExecute()`: 后台预加载
- `QuickSwitchNodeAsync()`: 快速切换到缓存结果
- 缓存统计和清理

**使用示例**:
```csharp
var orchestrator = new WorkflowExecutionOrchestrator(
    nodeResultManager, 
    maxCacheSize: 50, 
    maxMemoryMB: 100);

// 执行并显示（带缓存）
var result = await orchestrator.ExecuteAndDisplayAsync(
    node, imageSourceId, inputMat, workflow);

// 预执行（后台）
orchestrator.PreExecute(node, imageSourceId, inputMat, workflow);

// 快速切换（缓存命中时<10ms）
bool success = orchestrator.QuickSwitchNodeAsync(node, imageSourceId);
```

## 性能预期

| 场景 | 优化前延迟 | 优化后延迟 | 提升幅度 |
|------|-----------|-----------|---------|
| 首次执行 | 50-150ms | 50-150ms | 无变化 |
| 缓存命中 | 50-150ms | <10ms | **80-90%** |
| 节点切换（缓存） | 100-200ms | <20ms | **80%** |

**内存开销**: 约50MB（缓存50张图像）

## 使用建议

### 1. 正常工作流执行
使用现有的`ExecuteRunWorkflow()`方法，协调器会自动优化。

### 2. 图像预览切换
在图像预览控件中，使用协调器的缓存优先策略：
```csharp
// 先尝试缓存
if (orchestrator.QuickSwitchNodeAsync(node, imageSourceId))
{
    // 缓存命中，直接显示
}
else
{
    // 缓存未命中，执行工作流
    await orchestrator.ExecuteAndDisplayAsync(...);
}
```

### 3. 后台预加载
在用户浏览图像列表时，预加载相邻图像：
```csharp
// 预加载下一张图像
orchestrator.PreExecute(node, nextImageId, nextMat, workflow);
```

### 4. 缓存管理
定期清理缓存（可选）：
```csharp
// 获取缓存统计
var (count, memoryBytes, memoryMB) = viewModel.GetCacheStatistics();

// 清空缓存
viewModel.ClearExecutionCache();
```

## 注意事项

1. **内存管理**: 大量图像缓存会占用内存，建议根据实际场景调整缓存大小
2. **缓存失效**: 当工作流参数变化时，需清空相关节点的缓存
3. **线程安全**: 所有组件已实现线程安全，但建议在UI线程调用显示相关方法
