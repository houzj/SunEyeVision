## 实施内容

### 一、框架层新增（Plugin.SDK）

#### 1. RegionParameterMode 枚举
**文件位置**: `src/Plugin.SDK/Execution/Parameters/RegionParameterMode.cs`

**功能**：
- 定义区域参数的两种模式：
  - `InspectionRegion` - 检测区域
  - `MaskRegion` - 屏蔽区域

**关键特性**：
- 使用 `JsonStringEnumConverter` 支持序列化
- 符合参数系统规范

#### 2. RegionCollectionParameter 类
**文件位置**: `src/Plugin.SDK/Execution/Parameters/RegionCollectionParameter.cs`

**功能**：
- 支持区域集合管理
- 支持常量模式和绑定模式
- 继承 `ParamValueBase`，与 `ParamValue<T>` 保持一致性

**关键特性**：
- `Regions` 属性：`ObservableCollection<RegionData>` 区域集合
- `Mode` 属性：`RegionParameterMode` 区域参数模式
- `BindingConfig` 属性：`ParamSetting` 绑定配置
- 实现了 `ParamValueBase` 的所有抽象属性：
  - `ObjectValue`
  - `BindingConfig`
  - `ValueSource`
  - `IsBinding`
- 提供便利方法：
  - `GetEnabledRegions()` - 获取所有启用的区域
  - `GetVisibleRegions()` - 获取所有可见的区域
  - `ClearRegions()` - 清空所有区域
  - `AddRegion(RegionData)` - 添加区域
  - `RemoveRegion(RegionData)` - 移除区域
  - `Clone()` - 克隆区域集合参数

#### 3. RegionParameters 基类
**文件位置**: `src/Plugin.SDK/Execution/Parameters/RegionParameters.cs`

**功能**：
- 提供检测区域和屏蔽区域的通用实现
- 所有需要区域管理的工具可以继承此基类

**关键特性**：
- `InspectionRegions` 属性：`RegionCollectionParameter` 检测区域
- `MaskRegions` 属性：`RegionCollectionParameter` 屏蔽区域
- `IsRegionEnabled` 属性：`ParamValue<bool>` 是否启用区域限制
- `GetEffectiveRegions()` 方法：计算有效区域（检测区域 - 屏蔽区域）
- 实现 `OnClone()` 方法，支持深拷贝
- 使用 `[JsonDerivedType]` 支持多态序列化

**后台代码修改**: `tools/SunEyeVision.Tool.Threshold/Views/ThresholdToolDebugControl.xaml.cs`

**修改内容**：
- 添加 `inspectionRegionEditor` 和 `maskRegionEditor` 字段
- 添加 `InitializeInspectionRegionEditor()` 方法
- 添加 `InitializeMaskRegionEditor()` 方法
- 修改 `SetMainImageControl()` 方法，同时设置两个区域编辑器
- 修改 `SetParameters()` 方法，调用两个区域编辑器的初始化方法

**关键代码**：
```csharp
// 设置主窗口ImageControl
public virtual void SetMainImageControl(ImageControl? imageControl)
{
    _mainImageControl = imageControl;
    
    // 设置检测区域编辑器
    if (inspectionRegionEditor != null && imageControl != null)
    {
        inspectionRegionEditor.SetMainImageControl(imageControl);
        PluginLogger.Info("已为检测区域编辑器设置主窗口ImageControl", "ThresholdTool");
    }
    
    // 设置屏蔽区域编辑器
    if (maskRegionEditor != null && imageControl != null)
    {
        maskRegionEditor.SetMainImageControl(imageControl);
        PluginLogger.Info("已为屏蔽区域编辑器设置主窗口ImageControl", "ThresholdTool");
    }
}

// 设置节点参数
public void SetParameters(ToolParameters parameters)
{
    if (parameters is ThresholdParameters thresholdParams)
    {
        Parameters = thresholdParams;
        
        // 初始化区域编辑器
        InitializeInspectionRegionEditor();
        InitializeMaskRegionEditor();
    }
}

// 初始化检测区域编辑器
private void InitializeInspectionRegionEditor()
{
    if (inspectionRegionEditor == null)
        return;
    
    PluginLogger.Info("初始化检测区域编辑器", "ThresholdTool");
    
    inspectionRegionEditor.Regions = Parameters.InspectionRegions.Regions;
    inspectionRegionEditor.SetMainImageControl(_mainImageControl);
    
    PluginLogger.Success($"检测区域编辑器初始化完成，区域数量: {Parameters.InspectionRegions.Regions.Count}", "ThresholdTool");
}

// 初始化屏蔽区域编辑器
private void InitializeMaskRegionEditor()
{
    if (maskRegionEditor == null)
        return;
    
    PluginLogger.Info("初始化屏蔽区域编辑器", "ThresholdTool");
    
    maskRegionEditor.Regions = Parameters.MaskRegions.Regions;
    maskRegionEditor.SetMainImageControl(_mainImageControl);
    
    PluginLogger.Success($"屏蔽区域编辑器初始化完成，区域数量: {Parameters.MaskRegions.Regions.Count}", "ThresholdTool");
}
```

### 三、数据流

```
用户在框架窗口中绘制区域
   ↓
工具的 UserControl (ThresholdToolDebugControl)
   ├── RegionEditorControl (检测区域编辑器)
   │   └── 绑定到 ThresholdParameters.InspectionRegions.Regions
   ├── RegionEditorControl (屏蔽区域编辑器)
   │   └── 绑定到 ThresholdParameters.MaskRegions.Regions
   └── 其他参数控件
       └── 绑定到 ThresholdParameters.Threshold
   ↓ WPF 数据绑定自动同步
ThresholdParameters (工具参数）
   ↓ 存储为节点参数
WorkflowNodeBase.Parameters (节点参数）
   ↓ 解决方案统一管理
Solution.Workflows.Nodes
   ↓ JSON序列化
Solution.json
   ↓ JSON反序列化
Solution.Workflows.Nodes
   ↓ 恢复参数
ThresholdTool.Execute(ThresholdParameters)
   ↓
算法层计算有效区域 = 检测区域 - 屏蔽区域
   ↓
在有效区域内执行阈值化
   ↓
输出结果
```

### 四、架构优势

| 层级 | 优势 |
|------|------|
| **框架层** | 通用能力，所有工具复用；支持常量/绑定模式；统一接口 |
| **工具层** | 简洁实现，继承基类；无需重复区域管理逻辑 |
| **UI层** | 统一的区域编辑器；用户无需重新学习；灵活的布局和交互 |
| **解决方案层** | 统一管理所有参数；自动序列化/反序列化 |
