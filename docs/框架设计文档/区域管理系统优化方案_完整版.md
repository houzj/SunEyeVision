# 区域管理系统优化方案（完整版）

## 📋 规则元数据
- **规则ID**: rule-014
- **优先级**: 🔴 Critical
- **适用范围**: 框架层（Plugin.SDK）+ 工具层
- **生效模式**: 总是应用
- **最后更新**: 2026-04-14

---

## 🎯 方案概述

本方案为 SunEyeVision 系统设计完整的区域管理系统，包括**框架层的通用抽象**和**工具层的具体实现**。通过新增框架层的区域参数类型，为所有工具提供统一的区域管理能力。

**核心设计原则**：
- ✅ 框架层提供通用能力，不包含具体工具命名
- ✅ 工具层继承框架层，添加工具特定参数
- ✅ 参数系统作为各层桥梁，支持序列化和UI绑定
- ✅ 遵循 rule-002：命名符合视觉软件风格（InspectionRegions、MaskRegions）
- ✅ 遵循 rule-010：使用 `[JsonDerivedType]` 支持多态序列化

### 1.2 工具层现状（Threshold工具）

```
Tools/SunEyeVision.Tool.Threshold/
├── ThresholdParameters.cs          # 阈值参数
├── ThresholdTool.cs              # 阈值工具
└── ThresholdResults.cs            # 阈值结果
```

**当前实现**：
- ✅ 继承 `ToolParameters`
- ✅ 使用 `ParamValue<int>` 包装参数
- ✅ 完整的参数验证逻辑
- ❌ **缺少区域参数支持**

### 1.3 问题诊断

#### 问题1：框架层缺少区域参数抽象
**症状**：
- 工具层需要区域参数时，必须自己在工具层定义 `List<RegionData>`
- 不同工具的区域参数设计不一致
- 没有统一的区域参数基类

**影响**：
- 代码重复：每个工具都要定义类似的区域参数
- UI集成复杂：属性面板无法统一处理区域参数
- 扩展性差：新增区域功能需要修改所有工具

#### 问题2：区域参数不支持绑定模式
**症状**：
- `RegionData` 仅支持用户直接绘制（ShapeParameters）
- 无法订阅其他节点的区域输出
- 无法通过参数绑定动态计算区域

**影响**：
- 工作流灵活性差：无法动态调整区域
- 跨工具协作困难：无法共享区域数据

#### 问题3：缺少检测区域/屏蔽区域的语义
**症状**：
- 框架层只有通用的 `RegionData` 概念
- 没有明确的检测区域（InspectionRegion）和屏蔽区域（MaskRegion）语义
- 有效区域计算逻辑分散在各工具中

**影响**：
- 命名不清晰：开发者不知道如何使用区域参数
- UI展示不友好：用户不清楚区域的作用

```
   ↓

   ↓

   ↓
4. ThresholdParameters.InspectionRegions
   ↓

   ↓
9. 在有效区域内执行阈值化
   ↓
```

### 3.2 新增：RegionCollectionParameter 类

**文件位置**：`src/Plugin.SDK/Execution/Parameters/RegionCollectionParameter.cs`

```csharp
using System;
using System.Collections.Generic;
using System.Collections.ObjectModel;
using System.Linq;
using System.Text.Json.Serialization;
using SunEyeVision.Plugin.SDK.Models;
using SunEyeVision.Plugin.SDK.UI.Controls.Region.Models;

namespace SunEyeVision.Plugin.SDK.Execution.Parameters
{
    /// <summary>
    /// 区域集合参数 - 支持常量模式和绑定模式
    /// </summary>
    /// <remarks>
    /// 核心设计：
    /// 1. 常量模式：用户直接绘制区域（通过RegionEditorControl）
    /// 2. 绑定模式：订阅其他节点的区域输出
    /// 3. 继承ParamValueBase，与ParamValue<T>保持一致性
    /// 
    /// 命名规范（rule-002）：
    /// - InspectionRegions：检测区域
    /// - MaskRegions：屏蔽区域
    /// 
    /// 使用示例：
    /// <code>
    /// // 方式1：常量模式（直接绘制）
    /// public RegionCollectionParameter InspectionRegions { get; set; } 
    ///     = new RegionCollectionParameter(RegionParameterMode.InspectionRegion);
    /// 
    /// // 方式2：绑定模式（订阅其他节点）
    /// public RegionCollectionParameter InspectionRegions { get; set; } 
    ///     = new RegionCollectionParameter(RegionParameterMode.InspectionRegion)
    ///     {
    ///         BindingConfig = new ParamSetting
    ///         {
    ///             SourceNodeId = "region_tool_id",
    ///             SourceProperty = "OutputRegions"
    ///         }
    ///     };
    /// </code>
    /// </remarks>
    public class RegionCollectionParameter : ParamValueBase
    {
        private readonly ObservableCollection<RegionData> _regions = new();
        private RegionParameterMode _mode;

        /// <summary>
        /// 区域集合（算法层直接使用）
        /// </summary>
        /// <remarks>
        /// 无论常量模式还是绑定模式，Value 都存储具体的区域列表。
        /// 算法层只需要读取 Value，不需要关心来源。
        /// </remarks>
        public ObservableCollection<RegionData> Regions
        {
            get => _regions;
        }

        /// <summary>
        /// 区域参数模式
        /// </summary>
        [JsonIgnore]
        public RegionParameterMode Mode => _mode;

        /// <summary>
        /// 参数值（基类非泛型版本）
        /// </summary>
        public override object? ObjectValue
        {
            get => _regions;
            set
            {
                if (value is ObservableCollection<RegionData> regionCollection)
                {
                    _regions.Clear();
                    foreach (var region in regionCollection)
                    {
                        _regions.Add(region);
                    }
                }
            }
        }

        /// <summary>
        /// 绑定配置（框架层管理）
        /// </summary>
        public override ParamSetting? BindingConfig
        {
            get => _bindingConfig;
            set
            {
                if (SetProperty(ref _bindingConfig, value))
                {
                    OnPropertyChanged(nameof(ValueSource));
                    OnPropertyChanged(nameof(IsBinding));
                }
            }
        }

        /// <summary>
        /// 值来源标识（用于调试显示）
        /// </summary>
        [JsonIgnore]
        public override string ValueSource
        {
            get
            {
                if (BindingConfig == null)
                    return $"常量 ({_regions.Count} 个区域)";

                if (BindingConfig.BindingType == BindingType.Binding)
                    return $"绑定: {BindingConfig.SourceNodeId}.{BindingConfig.SourceProperty}";

                return "未知";
            }
        }

        /// <summary>
        /// 是否为绑定模式
        /// </summary>
        [JsonIgnore]
        public override bool IsBinding => BindingConfig != null && BindingConfig.BindingType == BindingType.Binding;

        /// <summary>
        /// 构造函数
        /// </summary>
        /// <param name="mode">区域参数模式</param>
        public RegionCollectionParameter(RegionParameterMode mode)
        {
            _mode = mode;
        }

        /// <summary>
        /// 添加区域
        /// </summary>
        public void AddRegion(RegionData region)
        {
            _regions.Add(region);
        }

        /// <summary>
        /// 移除区域
        /// </summary>
        public void RemoveRegion(RegionData region)
        {
            _regions.Remove(region);
        }

        /// <summary>
        /// 清空所有区域
        /// </summary>
        public void ClearRegions()
        {
            _regions.Clear();
        }

        /// <summary>
        /// 获取启用的区域列表
        /// </summary>
        public List<RegionData> GetEnabledRegions()
        {
            return _regions.Where(r => r.IsEnabled).ToList();
        }

        /// <summary>
        /// 克隆
        /// </summary>
        public RegionCollectionParameter Clone()
        {
            var clone = new RegionCollectionParameter(_mode)
            {
                _bindingConfig = _bindingConfig?.Clone()
            };

            foreach (var region in _regions)
            {
                clone._regions.Add((RegionData)region.Clone());
            }

            return clone;
        }
    }
}
```

**文件位置**：`src/Plugin.SDK/Execution/Parameters/RegionParameters.cs`

```csharp
using System.Collections.Generic;
using System.Text.Json.Serialization;
using SunEyeVision.Plugin.SDK.UI.Controls.Region.Models;

namespace SunEyeVision.Plugin.SDK.Execution.Parameters
{
    /// <summary>
    /// 区域参数基类 - 提供检测区域和屏蔽区域的通用支持
    /// </summary>
    /// <remarks>
    /// 框架层提供通用能力，工具层继承此基类并添加工具特定参数。
    /// 
    /// 使用示例：
    /// <code>
    /// public class ThresholdParameters : RegionParameters
    /// {
    ///     public ParamValue<int> Threshold { get; set; }
    /// }
    /// </code>
    /// </remarks>
    [JsonDerivedType(typeof(RegionParameters), "RegionParameters")]
    public class RegionParameters : ToolParameters
    {
        private RegionCollectionParameter _inspectionRegions;
        private RegionCollectionParameter _maskRegions;

        /// <summary>
        /// 构造函数
        /// </summary>
        public RegionParameters()
        {
            _inspectionRegions = new RegionCollectionParameter(RegionParameterMode.InspectionRegion);
            _maskRegions = new RegionCollectionParameter(RegionParameterMode.MaskRegion);
        }

        /// <summary>
        /// 检测区域（用户绘制的需要处理的区域）
        /// </summary>
        /// <remarks>
        /// 支持多区域，支持圆形、矩形、多边形等形状。
        /// 有效区域 = 检测区域 - 屏蔽区域（由算法层计算）。
        /// </remarks>
        [JsonIgnore]
        public RegionCollectionParameter InspectionRegions
        {
            get => _inspectionRegions;
            set => SetProperty(ref _inspectionRegions, value);
        }

        /// <summary>
        /// 屏蔽区域（用户绘制的不需要处理的区域）
        /// </summary>
        /// <remarks>
        /// 支持多区域，支持圆形、矩形、多边形等形状。
        /// </remarks>
        [JsonIgnore]
        public RegionCollectionParameter MaskRegions
        {
            get => _maskRegions;
            set => SetProperty(ref _maskRegions, value);
        }

        /// <summary>
        /// 是否启用区域限制
        /// </summary>
        /// <remarks>
        /// 如果为 false，忽略所有区域参数，对整个图像进行处理。
        /// </remarks>
        public ParamValue<bool> IsRegionEnabled { get; set; } 
            = ParamValue<bool>.CreateConstant(false, "启用区域限制");
    }
}
```

```csharp
namespace SunEyeVision.Plugins.Threshold

    /// <summary>
    /// 阈值工具参数
    /// </summary>
    [JsonDerivedType(typeof(ThresholdParameters), "Threshold")]
    public class ThresholdParameters : RegionParameters
    {
        // ============================================================
        // 继承自 RegionParameters 的参数：
        // - InspectionRegions (检测区域)
        // - MaskRegions (屏蔽区域)
        // - IsRegionEnabled (是否启用区域限制)
        // ============================================================

        // ============================================================
        // 工具特定参数
        // ============================================================

        private ParamValue<int> _threshold = ParamValue<int>.CreateConstant(128, "阈值");

        /// <summary>
        /// 阈值
        /// </summary>
        [ParameterRange(0, 255)]
        [ParameterDisplay(DisplayName = "阈值")]
        public ParamValue<int> Threshold
        {
            get => _threshold;
            set => SetProperty(ref _threshold, value);
        }

        /// <summary>
        /// 参数验证
        /// </summary>
        public override ValidationResult Validate()
        {
            var result = base.Validate();

            // 验证阈值范围
            if (Threshold.Value < 0 || Threshold.Value > 255)
            {
                result.AddError("阈值必须在 0-255 范围内");
            }

            // 验证区域参数
            if (IsRegionEnabled.Value)
            {
                if (InspectionRegions.GetEnabledRegions().Count == 0)
                {
                    result.AddError("启用区域限制时，必须至少设置一个检测区域");
                }
            }

            return result;
        }
    }
}
```

```csharp
            try
            {
                // 1. 判断是否启用区域限制
                if (!parameters.IsRegionEnabled.Value)
                {
                    logger?.LogInfo("区域限制未启用，对整个图像进行处理");

                    // 对整个图像进行阈值化
                    var result = ApplyThresholdToWholeImage(
                        parameters.Threshold.Value
                    );

                    return result;
```

```csharp
using SunEyeVision.Plugin.SDK.Execution.Parameters;
using SunEyeVision.Plugin.SDK.UI.Controls.Region.Models;

namespace SunEyeVision.Plugins.Threshold.Views
{
    /// <summary>
    /// 阈值工具调试控件
    /// </summary>
    public partial class ThresholdDebugControl
    {
        private ThresholdParameters _parameters;

        /// <summary>
        /// 构造函数
        /// </summary>
        public ThresholdDebugControl(ThresholdParameters parameters)
        {
            InitializeComponent();
            _parameters = parameters;

            // 集成区域编辑器
            IntegrateRegionEditor();
        }

        /// <summary>
        /// 集成区域编辑器
        /// </summary>
        private void IntegrateRegionEditor()
        {
            // 1. 设置检测区域编辑器
            InspectionRegionEditor.DataContext = new RegionEditorViewModel
            {
                Regions = _parameters.InspectionRegions.Regions,
                Mode = RegionEditorMode.Drawing
            };

            // 2. 设置屏蔽区域编辑器
            MaskRegionEditor.DataContext = new RegionEditorViewModel
            {
                Regions = _parameters.MaskRegions.Regions,
                Mode = RegionEditorMode.Drawing
            };

            // 3. 监听区域变化
            _parameters.InspectionRegions.Regions.CollectionChanged += 
                (s, e) => OnRegionsChanged();
            _parameters.MaskRegions.Regions.CollectionChanged += 
                (s, e) => OnRegionsChanged();
        }

        /// <summary>
        /// 区域变化处理
        /// </summary>
```

## 九、向后兼容性

### 9.1 兼容性设计

#### 策略1：默认不启用区域限制
```csharp
public ParamValue<bool> IsRegionEnabled { get; set; } 
    = ParamValue<bool>.CreateConstant(false, "启用区域限制");
```
- 默认值为 `false`
- 旧版本项目文件加载时，自动使用默认值
- 不影响现有工作流

#### 策略2：参数验证宽松
```csharp
public override ValidationResult Validate()
{
    var result = base.Validate();

    // 仅在启用区域限制时才验证区域参数
    if (IsRegionEnabled.Value)
    {
        if (InspectionRegions.GetEnabledRegions().Count == 0)
        {
            result.AddError("启用区域限制时，必须至少设置一个检测区域");
        }
    }

    return result;
}
```

### 9.2 数据迁移方案

**旧版本文件格式**：
```json
{
  "$type": "Threshold",
  "Threshold": 128
}
```

**新版本文件格式**：
```json
{
  "$type": "Threshold",
  "Threshold": 128,
  "IsRegionEnabled": false,
  "InspectionRegions": {
    "Regions": [],
    "BindingConfig": null
  },
  "MaskRegions": {
    "Regions": [],
    "BindingConfig": null
  }
}
```

**迁移策略**：
1. 加载旧版本文件时，缺失的区域参数自动使用默认值
2. 保存新版本文件时，自动包含完整的新参数
3. 用户首次使用时，看不到变化（默认不启用）

## 十、总结

### 10.1 关键改进点

#### 1. 框架层通用能力
- ✅ **RegionParameterMode**：区域参数模式枚举
- ✅ **RegionCollectionParameter**：区域集合参数类型（支持常量/绑定）
- ✅ **RegionParameters**：区域参数基类（可选）
- ✅ 统一的区域管理抽象，所有工具可复用

#### 2. 工具层简洁实现
- ✅ 继承 `RegionParameters`，自动获得区域参数
- ✅ 只需添加工具特定参数（如 Threshold）
- ✅ 无需重复实现区域管理逻辑

#### 3. 命名清晰规范
- ✅ **InspectionRegions**：检测区域（符合视觉软件习惯）
- ✅ **MaskRegions**：屏蔽区域（符合视觉软件习惯）
- ✅ **IsRegionEnabled**：是否启用区域限制

#### 4. 扩展性强
- ✅ 支持多区域
- ✅ 支持多种形状（矩形、圆形、多边形等）
- ✅ 支持常量模式和绑定模式
- ✅ 有效区域由算法层计算，不存储

### 10.2 符合的规则

- ✅ **rule-001**: 属性更改通知统一规范（继承 ObservableObject）
- ✅ **rule-002**: 命名规范（PascalCase，符合视觉软件习惯）
- ✅ **rule-003**: 日志系统使用规范（使用 parameters.LogInfo 等）
- ✅ **rule-010**: 方案系统实现规范（使用 `[JsonDerivedType]`）
- ✅ **rule-012**: 参数系统约束条件（参数系统作为各层桥梁）

### 10.3 架构优势

#### 框架层
- **通用性**：提供所有工具通用的区域参数类型
- **可扩展**：支持常量模式和绑定模式
- **一致性**：统一的区域管理接口

#### 工具层
- **简洁性**：继承基类，自动获得区域参数
- **灵活性**：可添加工具特定参数
- **可维护性**：区域管理逻辑集中在框架层

#### UI层
- **统一性**：所有工具使用相同的区域编辑器
- **易用性**：用户无需重新学习不同工具的区域编辑方式
- **扩展性**：预留屏蔽区域控件扩展点
