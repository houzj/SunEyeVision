## 🎯 设计目标

### 核心需求

1. **执行流与数据流完全分离**
   - 检测程序（执行逻辑）与检测配方（数据配置）独立管理
   - 支持独立升级和配置

2. **简化设备绑定**
   - 合并设备绑定到配方中
   - 参数一致即视为同一设备

3. **快速切换**
   - 产品切换 < 1 秒
   - 配方切换 < 1 秒
   - 支持最近使用列表

4. **批量升级**
   - 100 台设备批量升级 < 5 分钟
   - 只需修改引用，无需逐个修改文件

## 🏗️ 架构设计

### 整体架构

```
Solution（解决方案）
    └── Project（项目）
            ├── InspectionProgram（检测程序 - 执行流）
            │   ├── Nodes（节点拓扑）
            │   ├── Connections（连接关系）
            │   └── DefaultParams（默认参数）
            └── InspectionRecipes（检测配方 - 数据流）
                ├── NodeParams（节点参数）
                ├── GlobalVariables（全局变量）
                └── Device（设备信息）
```

### 层次关系

| 层级 | 名称 | 职责 | 数量 |
|------|------|------|------|
| L1 | Solution | 目录层级，不需要索引文件 | N 个 |
| L2 | Project | 一个产品对应一个项目 | 1 个产品 = 1 个项目 |
| L3 | Program | 执行流，检测逻辑 | 1 个项目 = 1 个程序 |
| L3 | Recipes | 数据流，参数配置 | 1 个项目 = N 个配方 |

## 📦 核心类设计

### 1. Project（项目）

**文件**: `src/Workflow/Project.cs`

**职责**: 一个项目对应一个产品的检测方案

**核心属性**:
```csharp
public class Project : ObservableObject
{
    public string Id { get; set; }
    public string Name { get; set; }
    public string ProductCode { get; set; }
    public string Description { get; set; }
    
    public InspectionProgram Program { get; set; }              // 执行流
    public ObservableCollection<InspectionRecipe> Recipes { get; set; }  // 配方列表
    
    public DateTime CreatedTime { get; set; }
    public DateTime ModifiedTime { get; set; }
}
```

**核心方法**:
- `AddRecipe(name, description)` - 添加配方
- `RemoveRecipe(recipeId)` - 移除配方
- `GetRecipe(recipeId)` - 获取配方
- `GetRecipeByName(name)` - 按名称获取配方
- `DuplicateRecipe(recipeId, newName)` - 复制配方
- `Validate()` - 验证项目配置
- `Clone()` - 克隆项目

**使用场景**:
```csharp
// 创建新产品项目
var project = projectManager.CreateProject("产品A100", "A100-2026-001", "高精度检测");

// 添加配方
project.AddRecipe("标准配方", "默认标准配方");
project.AddRecipe("产线1配方", "产线1专用配方");

// 切换项目
projectManager.SetCurrentProject(projectId, "标准配方");
```

### 2. InspectionProgram（检测程序）

**文件**: `src/Workflow/InspectionProgram.cs`

**职责**: 管理检测的执行逻辑（执行流）

**核心属性**:
```csharp
public class InspectionProgram : ObservableObject
{
    public string Id { get; set; }
    public string Name { get; set; }
    public string Version { get; set; }
    public string Description { get; set; }
    
    public List<ProgramNode> Nodes { get; set; }                    // 节点列表
    public List<ProgramConnection> Connections { get; set; }        // 连接关系
    public Dictionary<string, ToolParameters> DefaultParams { get; set; }  // 默认参数
    
    public DateTime CreatedTime { get; set; }
    public DateTime ModifiedTime { get; set; }
}
```

**节点定义**:
```csharp
public class ProgramNode
{
    public string Id { get; set; }
    public string Name { get; set; }
    public string Type { get; set; }
    public string? AlgorithmType { get; set; }
    public NodePosition Position { get; set; }
    public bool IsEnabled { get; set; }
    public ToolParameters? DefaultParameters { get; set; }
}
```

**连接定义**:
```csharp
public class ProgramConnection
{
    public string SourceNodeId { get; set; }
    public string SourcePort { get; set; }
    public string TargetNodeId { get; set; }
    public string TargetPort { get; set; }
}
```

**核心方法**:
- `AddNode(node)` - 添加节点
- `RemoveNode(nodeId)` - 移除节点
- `ConnectNodes(sourceId, sourcePort, targetId, targetPort)` - 连接节点
- `DisconnectNodes(sourceId, targetId)` - 断开连接
- `UpdateVersion(newVersion)` - 更新版本
- `Clone()` - 克隆程序

**使用场景**:
```csharp
// 创建节点
var node1 = new ProgramNode
{
    Name = "图像采集",
    Type = "Input",
    AlgorithmType = "ImageCapture",
    Position = new NodePosition { X = 100, Y = 100 }
};

var node2 = new ProgramNode
{
    Name = "阈值分割",
    Type = "Processing",
    AlgorithmType = "Threshold",
    Position = new NodePosition { X = 300, Y = 100 }
};

// 添加节点
program.AddNode(node1);
program.AddNode(node2);

// 连接节点
program.ConnectNodes(node1.Id, "Output", node2.Id, "Input");
```

### 3. InspectionRecipe（检测配方）

**文件**: `src/Workflow/InspectionRecipe.cs`

**职责**: 管理检测的数据配置和设备信息（数据流）

**核心属性**:
```csharp
public class InspectionRecipe : ObservableObject
{
    public string Id { get; set; }
    public string Name { get; set; }
    public string ProjectId { get; set; }
    public string Description { get; set; }
    
    public DeviceInfo? Device { get; set; }                         // 设备信息（合并）
    public Dictionary<string, ToolParameters> NodeParams { get; set; }  // 节点参数
    public Dictionary<string, GlobalVariable> GlobalVariables { get; set; }  // 全局变量
    
    public DateTime CreatedTime { get; set; }
    public DateTime ModifiedTime { get; set; }
}
```

**核心方法**:
- `SaveNodeParams(nodeId, parameters)` - 保存节点参数
- `GetNodeParams(nodeId)` - 获取节点参数
- `RemoveNodeParams(nodeId)` - 移除节点参数
- `ClearParams()` - 清空参数
- `SetGlobalVariable(name, value, type, description)` - 设置全局变量
- `GetGlobalVariable<T>(name)` - 获取全局变量
- `RemoveGlobalVariable(name)` - 移除全局变量
- `Clone()` - 克隆配方

**设备绑定合并原则**:
```
参数一致 = 同一设备

配方合并了设备绑定：
- 设备信息直接包含在配方中
- 参数相同即视为同一设备
- 简化了配置流程
```

**使用场景**:
```csharp
// 创建配方
var recipe = project.AddRecipe("产线1配方", "产线1专用");

// 设置节点参数
var thresholdParams = new ThresholdParameters { Threshold = 128 };
recipe.SaveNodeParams(nodeId, thresholdParams);

// 设置全局变量
recipe.SetGlobalVariable("ExposureTime", 100, "Double", "曝光时间");
recipe.SetGlobalVariable("Gain", 2.5, "Double", "增益");

// 设置设备信息
recipe.Device = new DeviceInfo
{
    DeviceId = "CAM_001",
    DeviceName = "产线1相机",
    DeviceType = "Camera",
    Manufacturer = "Basler",
    Model = "acA1920-40gm"
};
```

**核心方法**:
- `SetCurrentProject(projectId, recipeName)` - 设置当前项目

### 5. RunContext（执行上下文）

**文件**: `src/Workflow/RunContext.cs`

**职责**: 运行时临时对象，组合 Program + Recipe

**核心属性**:
```csharp
public class RunContext
{
    public InspectionProgram Program { get; set; }      // 检测程序
    public InspectionRecipe Recipe { get; set; }        // 检测配方
    
    public DateTime CreatedTime { get; }
    public string ExecutionId { get; }
}
```

**核心方法**:
- `GetNodeParameters(nodeId)` - 获取节点参数（优先配方参数，其次程序默认参数）
- `GetNodeParameters<T>(nodeId)` - 获取指定类型的节点参数
- `GetGlobalVariable<T>(name)` - 获取全局变量
- `GetAllNodeIds()` - 获取所有节点ID
- `GetEnabledNodes()` - 获取启用的节点
- `GetInputConnections(nodeId)` - 获取输入连接
- `GetOutputConnections(nodeId)` - 获取输出连接
- `Validate()` - 验证执行上下文
- `GetStatistics()` - 获取统计信息

**参数查询优先级**:
```
1. 配方中的节点参数（Recipe.NodeParams）
2. 程序中的默认参数（Program.DefaultParams）
```

**使用场景**:
```csharp
// 创建执行上下文
var context = RunContext.Create(project, recipe);

// 获取节点参数
var params = context.GetNodeParameters(nodeId);

// 执行检测
foreach (var node in context.GetEnabledNodes())
{
    var nodeParams = context.GetNodeParameters(node.Id);
    ExecuteNode(node, nodeParams);
}
```

## 🔄 与旧架构对比

### 架构对比

| 方面 | 旧架构 | 新架构 | 改进 |
|------|--------|--------|------|
| **层次结构** | Solution → Workflows + Recipes | Project → Program + Recipes | 层次更清晰，归属关系明确 |
| **设备绑定** | Recipe + DeviceBinding 分离 | InspectionRecipe 合并 | 简化配置，避免繁琐对应 |
| **执行上下文** | ExecutionContext | RunContext | 命名更准确，职责更明确 |
| **管理器** | SolutionManager | ProjectManager | 职责更清晰，方法更直观 |
| **配置持久化** | 无统一管理 | RuntimeConfig | 支持状态恢复 |

### 性能对比

| 操作 | 旧架构 | 新架构 | 提升 |
|------|--------|--------|------|
| **产品切换** | 2-4 秒 | < 1 秒 | 2-4 倍 |
| **配方切换** | 1-2 秒 | < 1 秒 | 1-2 倍 |
| **批量升级（100台）** | 2-4 小时 | < 5 分钟 | 24-48 倍 |
| **状态恢复** | 不支持 | 即时恢复 | 用户体验大幅提升 |

```csharp
// 旧方式：需要多个步骤
var solution = solutionManager.LoadSolution(solutionId);
var workflow = solutionManager.LoadWorkflow(binding.WorkflowRef);
var config = solutionManager.LoadDataConfig(binding.DataConfigRef);
var context = new RuntimeContext { Workflow = workflow, DataConfig = config };
```

#### 新架构：产品切换
```csharp
// 新方式：一行代码
projectManager.SetCurrentProject(projectId, "标准配方");
var context = projectManager.CreateRunContext();
```

## 🚀 迁移指南

### 从旧架构迁移

#### 自动迁移

```csharp
// 使用 ProjectManager 的迁移方法
var solution = solutionManager.LoadSolution(solutionId);
var project = projectManager.MigrateFromSolution(solution);

// 自动完成：
// 1. Solution → Project
// 2. WorkflowDefinition → InspectionProgram
// 3. Recipe → InspectionRecipe（合并设备绑定）
// 4. GlobalVariables → InspectionRecipe.GlobalVariables
```

#### 手动迁移步骤

1. **备份现有数据**
   ```bash
   cp -r solutions/ solutions_backup/
   ```

2. **运行迁移脚本**
   ```csharp
   var projectManager = new ProjectManager("solutions_new/");
   var solutionManager = new SolutionManager("solutions/");

   foreach (var solution in solutionManager.GetAllSolutions())
   {
       projectManager.MigrateFromSolution(solution);
   }
   ```

3. **验证迁移结果**
   ```csharp
   foreach (var project in projectManager.GetAllProjects())
   {
       if (!project.Validate())
       {
           LogError($"项目验证失败: {project.Name}");
       }
   }
   ```

4. **切换到新架构**
   ```bash
   mv solutions/ solutions_old/
   mv solutions_new/ solutions/
   ```

### 数据映射表

| 旧架构 | 新架构 | 说明 |
|--------|--------|------|
| `Solution.Id` | `Project.Id` | ID 保持不变 |
| `Solution.Name` | `Project.Name` | 名称保持不变 |
| `Solution.Description` | `Project.Description` | 描述保持不变 |
| `Solution.Workflows[0]` | `Project.Program` | 第一个工作流作为程序 |
| `Solution.Recipes` | `Project.Recipes` | 配方列表 |
| `Solution.GlobalVariables` | `Project.Recipes[0].GlobalVariables` | 全局变量移到第一个配方 |
| `Recipe.ParameterMappings` | `InspectionRecipe.NodeParams` | 参数映射重命名 |
| `RuntimeBinding.DeviceId` | `InspectionRecipe.Device.DeviceId` | 设备ID |
| `RuntimeBinding.WorkflowRef` | 删除 | 不再需要引用 |

## 📊 性能优化

### 文件加载优化

1. **延迟加载**
   - 项目列表只加载元数据
   - 程序和配方在需要时才加载

2. **缓存机制**
   - 已加载的项目缓存在内存中
   - 避免重复加载

3. **增量保存**
   - 只保存修改的部分
   - 避免全量保存

### 内存优化

1. **参数共享**
   - 相同参数实例可以共享
   - 减少内存占用

2. **按需加载**
   - 大型配方按需加载
   - 不使用的配方不加载

## ✅ 设计原则总结

1. **单一职责**
   - Program 管理执行逻辑
   - Recipe 管理数据配置
   - 各司其职，清晰明确

2. **开闭原则**
   - 对扩展开放（插件系统）
   - 对修改封闭（核心逻辑稳定）

3. **依赖倒置**
   - 依赖于抽象接口
   - 不依赖具体实现

4. **接口隔离**
   - 接口职责单一
   - 避免臃肿接口

5. **最小知识原则**
   - 模块间松耦合
   - 减少依赖关系
