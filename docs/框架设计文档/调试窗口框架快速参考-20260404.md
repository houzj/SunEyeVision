# 调试窗口框架快速参考

## 🚀 快速开始

### 1. 工具实现（3步）

```csharp
// Step 1: 添加 Tool 特性
[Tool(
    id: "my_tool",
    displayName: "我的工具",
    Category = "图像处理",
    HasDebugWindow = true  // 启用调试窗口
)]
public class MyTool : IToolPlugin<MyParameters, MyResults>
{
    // Step 2: 实现 Run 方法
    public MyResults Run(Mat image, MyParameters parameters)
    {
        // 执行逻辑
    }
    
    // Step 3: 实现 CreateDebugControl 方法
    public FrameworkElement? CreateDebugControl()
    {
        return new MyToolDebugControl();
    }
}
```

### 2. 调试控件实现（2步）

```csharp
// Step 1: 创建 UserControl

{
    // 直接引用参数（零拷贝）

    public MyToolDebugControl()
    {
        InitializeComponent();
        // 初始化UI
    }

    public event EventHandler? ConfirmClicked;

```

## 📋 窗口样式配置

### 节点级配置（优先级最高）

```csharp
// 在 WorkflowNode 中设置
node.DebugWindowStyle = DebugWindowStyle.Custom;
```

### 工具级配置

```csharp
[Tool(..., DebugWindowStyle = DebugWindowStyle.Custom)]
public class MyTool : IToolPlugin { }
```

### 窗口类型

| 样式 | 说明 | 适用场景 |
|------|------|----------|
| `None` | 无窗口 | 纯数据流节点 |
| `Default` | 标准窗口 | 大多数工具 |
| `Custom` | 自定义窗口 | 特殊视觉效果 |

## 🔄 工作流程（6步）

```
1. 数据获取 → context.Node = node
2. 元数据获取 → ToolRegistry.GetToolMetadata()
3. 窗口类型决策 → 节点级 → 工具级 → 全局默认
4. 窗口类型判断 → None 则返回
5. 内容创建 → node.CreateInstance() → tool.CreateDebugControl()
6. 策略执行 → 创建窗口并显示
```

## ⚠️ 常见问题

### Q1: 为什么不继承 BaseToolDebugControl？

**A**: 新架构采用普通 UserControl，不再需要基类：
- ✅ 简化设计，减少依赖
- ✅ 零拷贝参数，性能更好
- ✅ 直接绑定，代码更清晰

### Q2: 如何访问节点参数？

**A**: 调试控件中直接引用：
```csharp
// 调试控件中
private ThresholdParameters _parameters = null!;

// 修改参数实时同步
_thresholdSlider.Value = _parameters.Threshold;
```

### Q3: 如何自定义窗口样式？

**A**: 两种方式：
1. 使用 `DebugWindowStyle.Custom` 创建无边框圆角窗口
2. 实现 `INodeOpenStrategy` 创建完全自定义的窗口

### Q4: 如何避免阻塞UI？

**A**: 调试控件的创建和初始化应该轻量级：
```csharp
// ✅ 正确：轻量级创建
public FrameworkElement? CreateDebugControl()
{
    return new MyToolDebugControl();
}

// ❌ 错误：耗时操作
public FrameworkElement? CreateDebugControl()
{
    var control = new MyToolDebugControl();
    control.LoadHeavyData(); // 不要这样做
    return control;
}
```
