## 1. 概述

### 1.1 背景

在工业视觉检测系统中，存在以下典型需求：

- **多产品切换**：同一条产线可能生产多种产品，每种产品需要不同的检测参数
- **参数版本管理**：产品升级或工艺调整时，需要保存不同版本的参数配置
- **快速部署**：新产线需要快速复用已有配置
- **数据隔离**：不同产品的检测数据需要独立存储和管理
- **批量升级**：100台设备需要同时升级工作流版本

### 1.2 设计目标

SunEyeVision 解决方案系统旨在提供：

1. **执行流与数据流分离**：工作流（执行逻辑）与数据配置（参数配置）完全解耦
2. **层次化管理**：WorkflowDefinition（工作流）→ DataConfiguration（数据配置）→ RuntimeBinding（运行时绑定）
3. **灵活绑定**：支持设备、工作流、数据配置的动态组合
4. **多配方支持**：单个数据配置支持多个配方组，满足不同检测需求
5. **全局变量**：跨节点共享的变量，支持运行时动态修改
6. **可靠持久化**：基于 JSON 的可读性持久化方案

## 3. 解决方案系统架构

### 3.1 类关系详解

```csharp
// ============================================================
// 执行流配置 - 工作流定义
// ============================================================

/// <summary>
/// 工作流定义（执行逻辑）
/// </summary>
public class WorkflowDefinition
{
    public string Id { get; set; }
    public string Name { get; set; }
    public string Version { get; set; }
    public string Description { get; set; }
    
    /// <summary>
    /// 节点拓扑结构
    /// </summary>
    public List<WorkflowNode> Nodes { get; set; }
    
    /// <summary>
    /// 连接关系
    /// </summary>
    public List<Connection> Connections { get; set; }
    
    /// <summary>
    /// 节点默认参数（模板，用于新建配方时的初始值）
    /// </summary>
    public Dictionary<string, ToolParameters> DefaultParams { get; set; }
    
    /// <summary>
    /// 创建时间
    /// </summary>
    public DateTime CreatedTime { get; set; }
    
    /// <summary>
    /// 修改时间
    /// </summary>
    public DateTime ModifiedTime { get; set; }
}

// ============================================================
// 数据流配置 - 数据配置
// ============================================================

/// <summary>
/// 数据配置（配方组 + 全局变量）
/// </summary>
public class DataConfiguration
{
    public string Id { get; set; }
    public string Name { get; set; }
    public string Description { get; set; }
    
    /// <summary>
    /// 配方组（支持多种检测模式）
    /// Key: 配方组名称 (default, precision, fast等)
    /// </summary>
    public Dictionary<string, RecipeGroup> RecipeGroups { get; set; }
    
    /// <summary>
    /// 全局变量
    /// </summary>
    public Dictionary<string, GlobalVariable> GlobalVariables { get; set; }
    
    /// <summary>
    /// 创建时间
    /// </summary>
    public DateTime CreatedTime { get; set; }
    
    /// <summary>
    /// 修改时间
    /// </summary>
    public DateTime ModifiedTime { get; set; }
}

/// <summary>
/// 配方组（一个产品可以有多个配方组，如标准/精密/快速）
/// </summary>
public class RecipeGroup
{
    public string Name { get; set; }
    public string Description { get; set; }
    
    /// <summary>
    /// 节点参数映射：NodeId -> ToolParameters
    /// </summary>
    public Dictionary<string, ToolParameters> NodeParams { get; set; }
    
    /// <summary>
    /// 创建时间
    /// </summary>
    public DateTime CreatedTime { get; set; }
    
    /// <summary>
    /// 修改时间
    /// </summary>
    public DateTime ModifiedTime { get; set; }
}

// ============================================================
// 运行时绑定
// ============================================================

/// <summary>
/// 运行时绑定（设备 → 工作流 + 数据配置）
/// </summary>
public class RuntimeBinding
{
    public string DeviceId { get; set; }
    public string DeviceName { get; set; }
    
    /// <summary>
    /// 工作流引用（相对路径或ID）
    /// </summary>
    public string WorkflowRef { get; set; }
    
    /// <summary>
    /// 数据配置引用（相对路径或ID）
    /// </summary>
    public string DataConfigRef { get; set; }
    
    /// <summary>
    /// 使用的配方组名称
    /// </summary>
    public string RecipeGroup { get; set; } = "default";
    
    /// <summary>
    /// 创建时间
    /// </summary>
    public DateTime CreatedTime { get; set; }
    
    /// <summary>
    /// 最后切换时间
    /// </summary>
    public DateTime LastSwitchTime { get; set; }
}
```

### 7.2 解耦优势对比表

| 场景 | 传统耦合架构 | 解耦架构 |
|------|-------------|---------|
| 100台设备升级工作流 | 修改100个Solution文件（2-4小时） | 修改1个binding引用或拷贝1个文件（< 5分钟） |
| 单台设备切换产品 | 需要重新配置参数（5-10分钟） | 只切换dataConfig引用（< 1秒） |
| 新产品导入 | 需要创建新Solution（2-4小时） | 只创建新的dataConfig（5-10分钟） |
| 工作流版本回退 | 需要恢复整个Solution（5-10分钟） | 只切换workflow版本引用（< 1秒） |
| 参数微调 | 修改嵌入的Recipe（影响整个文件） | 修改独立的dataConfig（不影响工作流） |
| 配方组切换 | 需要修改配方参数（2-5分钟） | 只切换recipeGroup引用（< 1秒） |
