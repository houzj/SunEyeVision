## 1. 插件架构概述

### 1.1 架构分层

```
┌─────────────────────────────────────────────────────┐
│   插件层（Plugins）                                  │
│   - 工具实现（IToolPlugin）                          │
│   - 调试控件（ToolDebugControlBase）                │
│   - 参数模型（ToolParameters）                       │
│   - 结果模型（ToolResults）                          │
└─────────────────────────────────────────────────────┘
              ↓ 依赖
┌─────────────────────────────────────────────────────┐
│   插件 SDK（Plugin.SDK）                             │
│   - 接口定义（IPlugin, IToolPlugin）                │
│   - 基类实现（ToolDebugControlBase）                │
│   - UI 控件（ImageControl, RegionEditor 等）        │
│   - 日志系统（PluginLogger）                         │
└─────────────────────────────────────────────────────┘
              ↓ 依赖
┌─────────────────────────────────────────────────────┐
│   核心层（Core）                                     │
│   - 工作流引擎（WorkflowEngine）                     │
│   - 执行上下文（ExecutionContext）                  │
│   - 数据管理（SolutionManager）                     │
└─────────────────────────────────────────────────────┘
```

### 1.2 插件类型

| 插件类型 | 接口 | 用途 |
|---------|------|------|
| **工具插件** | `IToolPlugin<TParams, TResults>` | 执行图像处理算法 |
| **工作流控制插件** | `IWorkflowControlPlugin` | 控制工作流执行流程 |
| **数据源插件** | `IDataSourcePlugin` | 提供外部数据输入 |

## 3. 工具插件开发

### 3.1 实现工具类

```csharp
using SunEyeVision.Plugin.SDK;
using SunEyeVision.Plugin.SDK.Execution.Parameters;
using SunEyeVision.Plugin.SDK.Execution.Results;
using SunEyeVision.Plugin.SDK.Metadata;

namespace SunEyeVision.Tool.YourTool
{
    /// <summary>
    /// 你的工具实现
    /// </summary>
    [Tool(
        id: "your_tool",
        name: "你的工具",
        category: "图像处理",
        description: "工具描述",
        version: "1.0.0",
        author: "Your Name")]
    public class YourTool : IToolPlugin<YourToolParameters, YourToolResults>
    {
        public string Id => "your_tool";
        public string Name => "你的工具";
        public string Category => "图像处理";

        /// <summary>
        /// 执行工具逻辑
        /// </summary>
        public YourToolResults Execute(YourToolParameters parameters, ExecutionContext context)
        {
            // 1. 验证参数
            if (parameters == null)
                throw new ArgumentNullException(nameof(parameters));

            // 2. 获取输入图像（从参数或上下文）
            var inputImage = GetInputImage(parameters, context);

            // 3. 执行处理逻辑
            var outputImage = ProcessImage(inputImage, parameters);

            // 4. 构建结果
            var results = new YourToolResults
            {
                OutputImage = outputImage,
                // 设置其他结果...
            };

            return results;
        }

        /// <summary>
        /// 验证参数
        /// </summary>
        public ValidationResult ValidateParameters(YourToolParameters parameters)
        {
            var errors = new List<string>();

            if (parameters.Threshold < 0 || parameters.Threshold > 255)
                errors.Add("阈值必须在 0-255 之间");

            return new ValidationResult(errors.Count == 0, errors);
        }

        private Mat GetInputImage(YourToolParameters parameters, ExecutionContext context)
        {
            // 从参数或上下文获取输入图像
            // ...
        }

        private Mat ProcessImage(Mat input, YourToolParameters parameters)
        {
            // 执行图像处理逻辑
            // ...
        }
    }
}
```

## 4. 调试控件开发

### 4.1 WPF XAML 根元素要求

**⚠️ 重要：由于 WPF 框架限制，不支持泛型类型作为 XAML 根元素。**

正确的使用方式：

#### ✅ 正确：XAML 根元素使用 UserControl

```xaml
<UserControl x:Class="SunEyeVision.Tool.YourTool.Views.YourToolDebugControl"
             xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
             xmlns:controls="clr-namespace:SunEyeVision.Plugin.SDK.UI.Controls;assembly=SunEyeVision.Plugin.SDK"
             xmlns:commands="clr-namespace:SunEyeVision.Plugin.SDK.Commands;assembly=SunEyeVision.Plugin.SDK"
             d:DesignHeight="600" d:DesignWidth="400">
    
    <UserControl.Resources>
        <!-- 资源定义 -->
    </UserControl.Resources>
    
    <Grid>
        <!-- UI 布局 -->
        <Button Content="执行"
                Command="commands:ToolCommands.Execute"/>
    </Grid>
</UserControl>
```

#### ✅ 正确：后台代码继承泛型基类

```csharp
using SunEyeVision.Plugin.SDK.UI.Controls;
using SunEyeVision.Plugin.SDK.Commands;

namespace SunEyeVision.Tool.YourTool.Views
{
    /// <summary>
    /// 你的工具调试控件
    /// </summary>
    public partial class YourToolDebugControl : ToolDebugControlBase<YourToolResults>
    {
        private YourToolParameters _parameters;

        public YourToolDebugControl()
        {
            InitializeComponent();
            // 基类已自动调用 SetupCommandBindings()
            
            // 初始化参数
            _parameters = new YourToolParameters();
            
            // 设置绑定
            SetupBindings();
        }

        /// <summary>
        /// 设置参数绑定
        /// </summary>
        private void SetupBindings()
        {
            var binding = new Binding("Threshold")
            {
                Source = _parameters,
                Mode = BindingMode.TwoWay
            };
            thresholdSlider.SetBinding(Slider.ValueProperty, binding);
        }

        /// <summary>
        /// 执行工具逻辑 - 基类调用
        /// </summary>
        protected override YourToolResults ExecuteTool()
        {
            // 实现执行逻辑
            // ...
        }

        /// <summary>
        /// 判断是否可执行 - 基类调用
        /// </summary>
        protected override bool CanExecuteTool()
        {
            return _parameters != null;
        }
    }
}
```

#### ❌ 错误：XAML 根元素使用泛型基类

```xaml
<!-- ❌ 不支持：泛型类型作为根元素 -->
<controls:ToolDebugControlBase x:TypeArguments="local:YourToolResults"
                                x:Class="YourToolDebugControl">
    <!-- 这会导致编译错误 -->
</controls:ToolDebugControlBase>
```

### 4.2 基类提供的功能

继承 `ToolDebugControlBase<TResults>` 后，自动获得：

| 功能 | 说明 |
|------|------|
| **路由事件** | `ToolExecutionCompletedEvent` - 工具执行完成事件 |
| **路由命令** | `Execute`, `Confirm`, `ContinuousExecute`, `ResetParameters` |
| **命令绑定** | `SetupCommandBindings()` - 自动设置命令绑定 |
| **执行逻辑** | `OnExecuteCommand()` - 自动封装执行流程 |
| **数据注入** | `SetCurrentNode()`, `SetDataProvider()` |
| **参数验证** | `ValidateParameters()` |
| **抽象方法** | `ExecuteTool()`, `CanExecuteTool()` |

### 4.3 编译时合并机制

```
┌─────────────────────────────────────────────────────┐
│ 第1步：XAML 编译器解析 XAML 文件                      │
│ - 读取 XAML 根元素                                   │
│ - 生成 .g.cs 文件（partial class）                  │
└─────────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│ 第2步：生成 .g.cs 文件                               │
│ - 基类被固定为 XAML 根元素类型（UserControl）        │
│ - 不考虑后台代码中的继承关系                         │
└─────────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────┐
│ 第3步：编译 partial class                           │
│ - .g.cs + .xaml.cs 合并                            │
│ - 编译器自动选择更具体的基类（ToolDebugControlBase）│
└─────────────────────────────────────────────────────┘
```

**示例：**

```csharp
// .g.cs（WPF 自动生成）
public partial class YourToolDebugControl : UserControl
{
    // ...
}

// .xaml.cs（手写代码）
public partial class YourToolDebugControl : ToolDebugControlBase<YourToolResults>
{
    // ...
}

// 最终编译结果：继承 ToolDebugControlBase<YourToolResults>
```

### 4.4 命令绑定

#### 方式1：声明式绑定（推荐）

```xaml
<UserControl x:Class="YourToolDebugControl"
             xmlns:commands="clr-namespace:SunEyeVision.Plugin.SDK.Commands;assembly=SunEyeVision.Plugin.SDK">
    
    <UserControl.CommandBindings>
        <CommandBinding Command="commands:ToolCommands.Execute"
                       Executed="OnExecuteCommand"
                       CanExecute="CanExecuteCommand"/>
        <CommandBinding Command="commands:ToolCommands.Confirm"
                       Executed="OnConfirmCommand"/>
    </UserControl.CommandBindings>
    
    <Button Content="执行"
            Command="commands:ToolCommands.Execute"/>
</UserControl>
```

#### 方式2：代码绑定（基类已自动设置）

```csharp
// 基类构造函数自动调用
protected virtual void SetupCommandBindings()
{
    CommandBindings.Add(new CommandBinding(
        ToolCommands.Execute,
        OnExecuteCommand,
        CanExecuteCommand));
    
    CommandBindings.Add(new CommandBinding(
        ToolCommands.Confirm,
        OnConfirmCommand,
        CanConfirmCommand));
    
    // ...
}
```

## 5. 参数系统

### 5.1 定义参数类

```csharp
using SunEyeVision.Plugin.SDK.Execution.Parameters;
using SunEyeVision.Plugin.SDK.Models;
using System.Text.Json.Serialization;

namespace SunEyeVision.Tool.YourTool.Models
{
    /// <summary>
    /// 你的工具参数
    /// </summary>
    [JsonDerivedType(typeof(YourToolParameters), "YourTool")]
    public class YourToolParameters : ToolParameters
    {
        private int _threshold;
        private double _minValue;
        private double _maxValue;

        /// <summary>
        /// 阈值
        /// </summary>
        public int Threshold
        {
            get => _threshold;
            set => SetProperty(ref _threshold, value, "阈值");
        }

        /// <summary>
        /// 最小值
        /// </summary>
        public double MinValue
        {
            get => _minValue;
            set => SetProperty(ref _minValue, value, "最小值");
        }

        /// <summary>
        /// 最大值
        /// </summary>
        public double MaxValue
        {
            get => _maxValue;
            set => SetProperty(ref _maxValue, value, "最大值");
        }

        /// <summary>
        /// 克隆参数
        /// </summary>
        public override ToolParameters Clone()
        {
            return new YourToolParameters
            {
                Threshold = Threshold,
                MinValue = MinValue,
                MaxValue = MaxValue
            };
        }
    }
}
```

### 5.2 参数特性

- **继承 `ToolParameters`**：获得属性通知、克隆、序列化等功能
- **添加 `[JsonDerivedType]` 特性**：支持多态序列化
- **使用 `SetProperty()` 方法**：触发属性通知并记录日志
- **实现 `Clone()` 方法**：支持线程安全的参数克隆

## 8. 最佳实践

### 8.1 命名规范

遵循项目的命名规范（rule-002）：

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| **类名** | PascalCase | `ThresholdTool` |
| **接口** | I + PascalCase | `IToolPlugin` |
| **方法** | PascalCase | `ExecuteTool()` |
| **属性** | PascalCase | `Threshold` |
| **私有字段** | _ + camelCase | `_threshold` |
| **常量** | UPPER_CASE | `MAX_THRESHOLD` |
| **布尔属性** | Is/Has/Can 前缀 | `IsEnabled` |

## 9. 常见问题

### 9.1 XAML 编译错误

**问题**：MC3074 - XML 命名空间中不存在标记 "ToolDebugControlBase.Resources"

**原因**：XAML 根元素使用了泛型基类

**解决**：
```xaml
<!-- ❌ 错误 -->
<controls:ToolDebugControlBase x:TypeArguments="local:YourToolResults">

<!-- ✅ 正确 -->
<UserControl x:Class="YourToolDebugControl">
```

### 9.2 参数序列化失败

**问题**：参数无法正确序列化到 JSON

**原因**：缺少 `[JsonDerivedType]` 特性

**解决**：
```csharp
// ✅ 添加特性
[JsonDerivedType(typeof(YourToolParameters), "YourTool")]
public class YourToolParameters : ToolParameters
{
    // ...
}
```

### 9.3 设计时不显示

**问题**：Visual Studio 设计器不显示控件

**原因**：构造函数中检测设计模式并 return

**解决**：
```csharp
// ❌ 移除设计时检测
if (DesignerProperties.GetIsInDesignMode(this))
    return;

// ✅ 使用 d:DesignHeight/d:DesignWidth
```

### 9.4 命令绑定失效

**问题**：按钮点击无反应

**原因**：命令绑定未正确设置

**解决**：
```xaml
<!-- ✅ 方式1：声明式绑定 -->
<UserControl.CommandBindings>
    <CommandBinding Command="commands:ToolCommands.Execute"
                   Executed="OnExecuteCommand"/>
</UserControl.CommandBindings>

<!-- ✅ 方式2：基类自动绑定（无需手动设置） -->
```
