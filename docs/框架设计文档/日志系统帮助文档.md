## 1. 日志系统概述

SunEyeVision 日志系统是一个高性能、多目标、异步处理的统一日志框架。

### 核心特性
- ✅ **高性能**：ConcurrentQueue 异步入队，写入延迟 < 0.01ms
- ✅ **多目标输出**：同时输出到文件、UI、调试窗口等
- ✅ **异步处理**：后台批量处理，不阻塞业务线程
- ✅ **线程安全**：支持多线程并发写入
- ✅ **内存安全**：队列容量限制，自动丢弃旧日志
- ✅ **批量优化**：定时批量刷新，减少 IO 操作
- ✅ **UI友好**：循环缓冲区，支持实时显示和搜索

### 架构层次
```
应用层
  ↓ PluginLogger（静态访问入口）
  ↓ ILogger（日志记录接口）
  ↓ VisionLogger（核心日志服务）
  ↓ ILogWriter（写入器接口）
  ↓ 写入器实现
    ├─ FileLogWriter（文件）
    ├─ UILogWriter（UI显示）
    ├─ ConsoleLogWriter（控制台）
    └─ DebugLogWriter（调试输出）
```

## 2. 日志级别说明

| 级别 | 说明 | 使用场景 | 图标 | 颜色 |
|------|------|----------|------|------|
| **Info** | 信息 | 状态变更、过程开始、中间状态 | 🔵 | 蓝色 #2196F3 |
| **Success** | 成功 | 任务完成、结果达标、操作完成 | ✅ | 绿色 #4CAF50 |
| **Warning** | 警告 | 有问题但程序能继续运行 | ⚠️ | 橙色 #FF9800 |
| **Error** | 错误 | 失败/异常，需要处理 | ❌ | 红色 #F44336 |

### 日志级别示例
```csharp
// Info - 一般信息
PluginLogger.Info("图像加载开始", "图像处理");

// Success - 成功操作
PluginLogger.Success("图像处理完成：检测到 5 个目标", "图像处理");

// Warning - 警告信息
PluginLogger.Warning("图像尺寸超出预期，将自动缩放", "图像处理");

// Error - 错误信息
PluginLogger.Error("图像加载失败：文件不存在", "图像处理", exception);
```

## 3. 核心组件

### 3.1 PluginLogger
**作用**：全局静态访问入口，提供便捷的日志记录方法。

**位置**：`src/Plugin.SDK/Logging/PluginLogger.cs`

**使用示例**：
```csharp
// 应用启动时配置
PluginLogger.SetProvider(VisionLogger.Instance);

// 在任何地方使用
PluginLogger.Info("日志消息", "来源");
```

### 3.2 VisionLogger
**作用**：核心日志服务，实现高性能异步日志处理。

**位置**：`src/Core/Services/Logging/VisionLogger.cs`

**核心特性**：
- 单例模式
- 异步队列处理（ConcurrentQueue）
- 多写入器支持
- 批量处理优化
- 内存限制保护

**性能指标**：
- 队列容量：50000 条
- 批量大小：200 条
- 批处理间隔：50ms
- 写入延迟：< 0.01ms

### 3.3 ILogWriter
**作用**：日志写入器接口，支持多种输出目标。

**位置**：`src/Plugin.SDK/Logging/ILogWriter.cs`

**实现类**：
- `FileLogWriter`：文件日志写入器
- `UILogWriter`：UI界面日志写入器
- `ConsoleLogWriter`：控制台日志写入器
- `DebugLogWriter`：调试输出日志写入器

### 3.4 LogEntry
**作用**：结构化日志条目，包含完整的日志信息。

**位置**：`src/Plugin.SDK/Logging/LogEntry.cs`

**核心属性**：
- `Id`：日志唯一标识符
- `Timestamp`：时间戳
- `Level`：日志级别
- `Message`：日志消息
- `Source`：来源标识
- `Exception`：异常信息
- `Properties`：扩展属性

## 4. 使用方法

### 4.1 在 ViewModel 中使用

```csharp
using SunEyeVision.Plugin.SDK.Logging;

public class MyViewModel : ViewModelBase
{
    private void ExecuteOperation()
    {
        // Info - 记录操作开始
        PluginLogger.Info("开始执行操作", "MyViewModel");

        try
        {
            // 业务逻辑
            var result = DoSomething();

            // Success - 记录成功
            PluginLogger.Success($"操作成功，结果：{result}", "MyViewModel");
        }
        catch (Exception ex)
        {
            // Error - 记录错误
            PluginLogger.Error("操作失败", "MyViewModel", ex);
        }
    }
}
```

## 5. 日志来源标识

### 5.1 三种获取 source 的方式

#### 方式一：手动传入字符串（最常用）

```csharp
// ✅ 使用预定义的 LogSources 常量
PluginLogger.Info("工作流启动", LogSources.WorkflowEngine);
PluginLogger.Info("图像加载完成", LogSources.ImageQueue);
PluginLogger.Info("插件加载成功", LogSources.PluginManager);

// ✅ 手动传入自定义字符串
PluginLogger.Info("节点执行开始", "阈值工具");
PluginLogger.Info("区域创建成功", "RegionEditor");
```

#### 方式二：继承 ObservableObject 并重写 GetLogSource（自动获取）

**适用于**：ViewModel、Model 等继承 ObservableObject 的类，特别是属性变更日志。

```csharp
public class MyViewModel : ObservableObject
{
    // 重写 GetLogSource 方法
    protected override string? GetLogSource()
    {
        return "MyViewModel"; // 或返回某个标识符
    }

    // 使用 SetProperty 时，source 会自动从 GetLogSource() 获取
    private double _threshold;
    public double Threshold
    {
        get => _threshold;
        set => SetProperty(ref _threshold, value, "阈值"); // 自动使用 GetLogSource() 的返回值
    }
}
```

**实际示例**：
```csharp
// ToolViewModelBase (Plugin.SDK) 的实现
public abstract class ToolViewModelBase : ObservableObject
{
    protected override string? GetLogSource()
    {
        return ToolName; // 返回工具名称
    }
}

// 使用
public class ThresholdToolViewModel : ToolViewModelBase
{
    private double _threshold;
    public double Threshold
    {
        get => _threshold;
        set => SetProperty(ref _threshold, value, "阈值");
        // 自动记录：参数 [阈值] 值已修改: 100.00 → 150.00，来源 = "阈值工具"
    }
}
```

#### 方式三：使用 ForSource 创建带预设来源的日志器（上下文绑定）

**适用于**：需要在一个作用域内多次使用相同来源的场景。

```csharp
// 创建带预设来源的日志器
var logger = VisionLogger.Instance.ForSource("MyService");

// 后续使用时无需传入 source
logger.Info("开始处理"); // source 自动为 "MyService"
logger.Success("处理完成");
logger.Error("处理失败", exception);
```

**实际示例 - 工作流执行中使用节点名称**：
```csharp
// WorkflowExecutionManager 的使用
var logger = VisionLogger.Instance.ForSource(workflow.Name);

// 在整个工作流执行过程中使用
logger.Info("工作流启动");
logger.Info("节点执行开始");
logger.Success("工作流完成");
```

### 5.2 工具（Tool）如何获得节点名称作为 source

**问题**：工具在执行时，如何知道当前节点的名称作为日志的 source？

**答案**：通过 `WorkflowNode.Name` 属性传递，路径如下：

```
UI 层 (WorkflowNodeModel)
    ↓ Name 属性
执行层 (WorkflowNode)
    ↓ Name 属性
工具层 (IToolPlugin.Run) ← 需要获取节点名称
```

#### 方案 A：通过 ToolMetadata.ToolName 获取（适用于属性变更日志）

**当前实现**（Plugin.SDK 的 ToolViewModelBase）：
```csharp
public abstract class ToolViewModelBase : ObservableObject
{
    protected override string? GetLogSource()
    {
        return ToolName; // 从 ToolMetadata 获取
    }
    
    // 属性变更日志自动使用 ToolName 作为 source
    private double _threshold;
    public double Threshold
    {
        get => _threshold;
        set => SetProperty(ref _threshold, value, "阈值");
        // 输出：参数 [阈值] 值已修改: 100 → 150，来源 = "阈值工具"
    }
}
```

**特点**：
- ✅ 适用于：属性变更日志
- ✅ 自动获取：无需手动传入
- ❌ 局限性：`ToolName` 来自 `ToolMetadata`，可能不等于节点名称

#### 方案 B：通过参数传递节点名称（推荐用于执行日志）

**工作流执行时传递节点名称**：

```csharp
// 1. WorkflowExecutionManager 创建节点
var logger = VisionLogger.Instance.ForSource(workflow.Name);
var algorithmNode = WorkflowNodeFactory.CreateAlgorithmNode(
    toolId,      // "threshold_tool"
    nodeId,      // "node_001"
    nodeName,    // "阈值工具_1" ← 这是用户设置的节点名称
    parameters
);

// 2. WorkflowNodeFactory 创建 AlgorithmNode
public static AlgorithmNode? CreateAlgorithmNode(
    string toolId,
    string nodeId,
    string nodeName,  // ← 节点名称
    ToolParameters? parameters = null,
    bool enableCaching = true,
    bool enableRetry = false)
{
    var algorithmNode = new AlgorithmNode(nodeId, nodeName, tool);
    return algorithmNode;
}

// 3. AlgorithmNode 保存节点名称
public AlgorithmNode(string id, string name, IToolPlugin tool)
    : base(id, name, NodeType.Algorithm)
{
    Tool = tool;
}

// 4. WorkflowNode.Name 保存节点名称
public class WorkflowNode
{
    public string Name { get; set; }  // ← 节点名称
}

// 5. 工具执行时获取节点名称
public AlgorithmResult Execute(Mat inputImage, IDictionary<string, ToolResults>? nodeResults)
{
    // 方式1：通过 WorkflowNode.Name 获取（推荐）
    var nodeName = Name;  // "阈值工具_1"
    
    // 方式2：通过 ILogger.ForSource 创建带节点名称的日志器
    var logger = Logger?.ForSource(nodeName);
    logger.Info("工具开始执行");
    
    // 方式3：手动传入节点名称
    PluginLogger.Info("工具开始执行", nodeName);
    
    // 执行工具
    var result = Tool.Run(inputImage, typedParams);
}
```

## 5.4 工具层获得上层信息作为 source 的完整示例

### 5.4.1 架构层次

```
UI 层 (ThresholdToolViewModel)
    ↓ 设置 ToolName = "阈值工具"
UI 层 (WorkflowNodeModel)
    ↓ Name 属性 = "阈值工具_1"
执行层 (AlgorithmNode)
    ↓ Name 属性 = "阈值工具_1"
    ↓ Parameters.NodeName = "阈值工具_1"
工具层 (IToolPlugin.Run) ← 获取节点名称作为 source
```

### 5.4.2 完整示例：工具如何获得节点名称作为 source

#### 步骤 1：在 ToolParameters 中添加节点信息

首先，我们需要在 `ToolParameters` 基类中添加节点信息的支持：

```csharp
// src/Plugin.SDK/Execution/Parameters/ToolParameters.cs
namespace SunEyeVision.Plugin.SDK.Execution.Parameters
{
    /// <summary>
    /// 工具参数基类
    /// </summary>
    public abstract class ToolParameters
    {
        /// <summary>
        /// 节点ID（工作流执行时自动设置）
        /// </summary>
        public string NodeId { get; set; } = string.Empty;
        
        /// <summary>
        /// 节点名称（工作流执行时自动设置）
        /// </summary>
        public string NodeName { get; set; } = string.Empty;
        
        /// <summary>
        /// 获取日志来源（优先使用节点名称）
        /// </summary>
        public string GetLogSource()
        {
            return !string.IsNullOrEmpty(NodeName) ? NodeName : "未命名工具";
        }
    }
}
```

#### 步骤 2：工作流执行时设置节点信息

在 `AlgorithmNode` 的 `Execute` 方法中，将节点信息传递给参数：

```csharp
// src/Workflow/AlgorithmNode.cs
public AlgorithmResult Execute(Mat inputImage, IDictionary<string, ToolResults>? nodeResults)
{
    OnBeforeExecute();

    try
    {
        var typedParams = GetTypedParameters();
        
        // ✅ 关键：设置节点信息到参数
        typedParams.NodeId = Id;
        typedParams.NodeName = Name;
        
        // 应用参数绑定解析
        if (ParameterBindings?.Count > 0 && nodeResults != null)
        {
            var resolver = new ParameterResolver();
            var applyResult = resolver.ApplyToParameters(typedParams, ParameterBindings, nodeResults);
            
            if (!applyResult.IsSuccess && applyResult.Errors.Count > 0)
            {
                var errors = string.Join("; ", applyResult.Errors);
                var result = AlgorithmResult.CreateError($"参数绑定解析失败: {errors}");
                OnAfterExecute(result);
                return result;
            }
        }
        
        // 执行工具
        var toolResult = Tool.Run(inputImage, typedParams);
        LastToolResult = toolResult;
        LastResult = ConvertToAlgorithmResult(toolResult);
    }
    else
    {
        LastResult = AlgorithmResult.CreateError("节点未配置工具实例");
    }

    OnAfterExecute(LastResult);
    return LastResult;
}
```

```csharp
[Tool("circle_find", "圆形检测",
    Description = "使用霍夫变换检测图像中的圆形",
    Icon = "⭕",
    Category = "形状识别")]
public class CircleFindTool : IToolPlugin<CircleFindParameters, CircleFindResults>
{
    public CircleFindResults Run(Mat image, CircleFindParameters parameters)
    {
        var nodeName = parameters.GetLogSource();
        
        // 预处理日志
        PluginLogger.Info("圆形检测工具开始执行", nodeName);
        PluginLogger.Info($"最小半径: {parameters.MinRadius}, 最大半径: {parameters.MaxRadius}", nodeName);
        PluginLogger.Info($"检测距离: {parameters.MinDistance}", nodeName);
        
        // 图像预处理
        Mat gray = new Mat();
        Mat blurred = new Mat();
        
        try
        {
            PluginLogger.Info("转换为灰度图像", nodeName);
            Cv2.CvtColor(image, gray, ColorConversionCodes.BGR2GRAY);
            
            PluginLogger.Info("应用高斯模糊降噪", nodeName);
            Cv2.GaussianBlur(gray, blurred, new Size(9, 9), 2, 2);
            
            // 检测圆形
            PluginLogger.Info("执行霍夫圆变换", nodeName);
            var circles = Cv2.HoughCircles(
                blurred,
                HoughModes.Gradient,
                parameters.AccumulatorResolution,
                parameters.MinDistance,
                parameters.CannyThreshold,
                parameters.AccumulatorThreshold,
                parameters.MinRadius,
                parameters.MaxRadius
            );
            
            var result = new CircleFindResults();
            result.Circles = circles;
            
            // 结果日志
            PluginLogger.Success($"检测完成，找到 {circles.Length} 个圆形", nodeName);
            PluginLogger.Success($"总耗时: {result.ExecutionTimeMs}ms", nodeName);
            
            if (circles.Length > 0)
            {
                PluginLogger.Info("圆形坐标:", nodeName);
                for (int i = 0; i < circles.Length; i++)
                {
                    var circle = circles[i];
                    PluginLogger.Info($"  圆形{i+1}: 中心=({circle.Center.X:F1}, {circle.Center.Y:F1}), 半径={circle.Radius:F1}", nodeName);
                }
            }
            
            return result;
        }
        finally
        {
            gray?.Dispose();
            blurred?.Dispose();
        }
    }
}
```

## 9. 常见问题

### Q1: 为什么日志没有输出？

**A**: 检查以下几点：
1. 是否调用了 `PluginLogger.SetProvider()`？
2. 是否添加了写入器？
3. 日志级别是否低于 `MinLevel`？
4. `IsEnabled` 是否为 `true`？

```csharp
// 正确的初始化流程
PluginLogger.SetProvider(VisionLogger.Instance);
VisionLogger.Instance.AddWriter(new UILogWriter());
VisionLogger.Instance.AddWriter(new FileLogWriter());

// 检查配置
Console.WriteLine($"IsEnabled: {VisionLogger.Instance.IsEnabled}");
Console.WriteLine($"MinLevel: {VisionLogger.Instance.MinLevel}");
Console.WriteLine($"WriterCount: {VisionLogger.Instance.WriterCount}");
```

### Q2: 如何减少日志输出？

**A**:
1. 提高 `MinLevel`：
```csharp
VisionLogger.Instance.SetMinLevel(LogLevel.Warning);
```

2. 禁用特定写入器：
```csharp
var writer = VisionLogger.Instance.GetWriterNames().FirstOrDefault();
VisionLogger.Instance.RemoveWriter(writer);
```

3. 设置写入器的最低级别：
```csharp
var uiWriter = new UILogWriter();
uiWriter.MinLevel = LogLevel.Warning; // 只输出 Warning 和 Error
```

### Q3: 如何查看特定来源的日志？

**A**: 在UI日志显示器中使用搜索功能：
```
[工作流引擎]    # 搜索工作流引擎的日志
[图像处理]    # 搜索图像处理的日志
[节点1]       # 搜索特定节点的日志
```

### Q4: 如何处理高并发日志？

**A**: 日志系统已经优化：
- 使用 `ConcurrentQueue` 异步入队
- 批量处理（200条/批）
- 自动内存保护（最大50000条）

如果仍然有性能问题：
1. 提高 `MinLevel` 减少日志量
2. 禁用不必要的写入器
3. 检查是否在循环中记录日志

### Q5: 如何导出日志？

**A**:
1. **文件日志**：直接复制日志文件（`logs/` 目录）
2. **UI日志**：使用UI的复制/导出功能
3. **自定义导出**：
```csharp
var uiWriter = new UILogWriter();
var logs = uiWriter.LogEntries;
var text = string.Join("\n", logs.Select(e => e.Format()));
File.WriteAllText("export.txt", text);
```
